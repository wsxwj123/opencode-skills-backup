# 07 收尾

目标：用户亲手确认真实效果，同步知识，清理现场。"GitHub 上发布了"不等于做完了。

## Step 1 ⏸ 卡点5：用户实测（按项目类型给用户可操作的验收方式）

| 类型 | 给用户什么 |
|---|---|
| 网页（已部署） | 线上地址，请用户亲自打开点一遍核心流程 |
| 程序/软件/脚本 | 具体运行命令（从安装到跑起来，照 README），请用户**在自己机器上实际跑一遍** |
| skill | 请用户开新会话真实调用一次，看产出是否符合预期 |

用户说不行 → 把问题记录下来，回 02 的修 bug 模式走一轮。
用户确认没问题 → 继续。

## Step 2：知识同步与复盘

1. 调用 `neat-freak` 技能：同步代码、文档（README/CLAUDE.md）、记忆三端一致
2. 复盘写进项目根 `LEARNINGS.md`：这次踩了什么坑、什么做法值得沿用（按自我进化规则：项目专属写本地，跨项目通用写 `~/.learnings/`）

## Step 3：清理（列清单，经用户确认后执行）

- 删除已合并的 feature 分支（本地 + 远程）
- 残留的 worktree 用 `git worktree remove` 清掉（严禁 `rm -rf`），再 `git worktree list` 确认干净
- 删除临时文件、临时数据
- `.devflow/` **保留**——它是项目档案，下次加功能/修 bug 从这里续接

## Step 4：结束报告

一段人话总结：做了什么、在哪能用、测了什么、遗留什么。流程结束。
