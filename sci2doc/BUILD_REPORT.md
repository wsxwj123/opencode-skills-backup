## 📝 核心工作流程

### A. 启动与状态检测
1. 读取 `project_state.json`
2. 判断新建 or 续写
3. 报告当前状态

### B. 初始化阶段
1. 收集论文信息（标题、作者、导师等）
2. 分析 SCI 论文（提取核心内容）
3. 生成详细大纲（精确到三级标题）
4. 创建项目文件夹结构

### C. 逐章生成阶段
1. 加载上下文（当前章节大纲+上一章结尾）
2. 内容生成（分小节，段落式写作）
3. 图片处理（占位符+说明文件）
4. 写入文件（Markdown → Docx with styles）
5. 字数核对（不达标立即扩展）
6. 更新状态（保存进度）
7. 用户确认（每章完成后）

### D. 后置处理与合并
1. 生成综述（≥5,000字，独立章节）
2. 整理参考文献（≥80篇）
3. 合并所有章节（自动生成目录）
4. 质量自检（生成报告）

### E. 续写与管理
- **查询状态**：显示完成进度和下一步
- **断点恢复**：从 `current_chapter_index` 继续
- **修改章节**：备份原文件后重新生成

## ⚠️ 重要约束

### 1. 工具调用规范
```python
# ❌ 错误示例（伪代码）
setup_page_format()
count_words()

# ✅ 正确示例（真实工具）
from docx import Document
doc = Document()
# 使用 python-docx 真实 API
```

### 2. 字数统计区分
- **正文字数**：不含综述、参考文献、目录
- **综述字数**：单独计算，不计入正文
- **目标**：正文≥50,000 + 综述≥5,000

### 3. 格式严格遵循
- **页边距**：上下 2.54cm，左 3.17cm，右 3.17cm
- **行距**：固定 20 磅
- **标题**：三级以内，不能跳级
- **段落**：无列表项，全部段落式
