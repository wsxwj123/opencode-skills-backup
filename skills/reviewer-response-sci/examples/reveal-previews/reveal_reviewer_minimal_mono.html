工作流总览

flowchart LR A[读取 Comments.docx] --> B[按 Reviewer / Major / Minor 切分] B --> C[每条 comment 原子化 JSON] C --> D[关联 manuscript / SI 位点] D --> E[生成中英回复与修稿建议] E --> F[渲染层级 HTML 成品] F --> G[质量门控与一致性检查]

每条 Comment 页面结构

  1. 审稿人意图理解：原始意见 + 中英文解读
  2. Response to Reviewer：中英双框上下排布，支持一键“复制”
  3. 修稿建议块：定位 section/句子、原文、改后英文、中文对照
  4. 修改说明：明确“添加 / 删除 / 修改”及理由
  5. 证据附件：文本 / 表格 / 图片（仅在需要时显示占位）

目录交互与阅读效率

层级目录

邮件总回复 → Reviewer → Major/Minor → Comment

支持 Reviewer 与 Section 折叠，聚焦当前评审人。

优先级视觉

目录叶子节点通过背景色区分重要级别。

避免符号噪音，提升快速扫描效率。

质量门控清单

  • 不编造数据，不超范围承诺
  • 回复语气礼貌、建设性、可审计
  • 修稿文本定位明确，可直接回填 Word
  • 目录、复制、折叠与响应式显示全部可用
