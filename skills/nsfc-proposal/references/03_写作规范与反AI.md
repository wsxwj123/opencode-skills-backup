# 03 写作规范与反AI

---

## 一、段落式叙事（最高优先级写作规范）

### 1.1 核心原则：讲故事，不做PPT

申请书的本质是向评审专家**讲述一个完整的科学故事**，而非展示一组信息条目。

**强制要求：**
- 全文采用**段落式阐述**，每个论点用完整段落展开
- **禁止**使用项目符号（•、-、*）展开论述
- **禁止**使用编号列表（1. 2. 3.）展开论述
- 段落之间通过逻辑过渡自然衔接，而非靠标号切换话题

**唯一例外（允许列表的场景）：**
- 年度计划：按年度分段，每年度内仍用段落描述
- P3_3 在研项目：项目信息本身是列表格式
- P3_4 结题项目成果目录：成果清单
- P4 其他说明：逐项回应模板要求
- 预算说明书中的三线表

### 1.2 段落结构要求

每个段落应具备：
- 主题句（段首，明确本段论点）
- 展开句（证据、数据、文献支撑）
- 过渡/总结句（衔接下段或小结）

段落长度：150-400字为宜，避免超长段落（>500字）或碎片段落（<80字）。

### 1.3 叙事逻辑模板

**立项依据推荐叙事线：**
```
宏观背景（领域重要性）
  → 聚焦到具体问题（研究对象）
    → 国内外研究现状（已知什么）
      → 关键空白/矛盾（不知道什么）
        → 本项目切入点（为什么我们能做）
          → 科学问题凝练（要回答什么）
```

**研究内容推荐叙事线：**
```
研究假说（基于立项依据的逻辑推演）
  → 研究目标（要做什么）
    → 研究内容（怎么组织）
      → 关键科学问题（要回答什么）
        → 技术路线（怎么做）
          → 创新点（新在哪里）
            → 年度计划（什么时候做）
```

## 二、反AI写作风格

### 2.1 禁用句式（一级强制）

以下句式**绝对禁止**出现在申请书任何位置：

| 禁用句式 | 原因 |
|---------|------|
| "不是...而是..." | 典型AI对比句式 |
| "不仅...而且..." | 典型AI递进句式 |
| "值得注意的是" / "需要指出的是" | AI口头禅 |
| "至关重要" / "举足轻重" / "不可或缺" | 空洞修饰 |
| "具有重要的理论意义和实践价值" | 万能废话 |
| "日益增长" / "蓬勃发展" / "方兴未艾" | 新闻体 |
| "深入探讨" / "系统研究" / "全面分析" | 空洞动词 |
| "综上所述" / "总而言之" | 机械过渡 |
| "在此基础上" / "鉴于此" | AI过渡模板 |

### 2.2 禁用修辞手法（一级强制）

| 禁用修辞 | 示例 | 替代方案 |
|---------|------|---------|
| 排比 | "...是A，是B，更是C" | 用一个精准的表述替代 |
| 比喻 | "...的桥梁"、"...的基石" | 直接陈述功能/作用 |
| 反问 | "难道不是...吗？" | 改为陈述句 |
| 设问 | "那么，XXX是什么？" | 直接阐述 |
| 夸张 | "革命性的"、"颠覆性的" | 用数据/事实说明程度 |

### 2.3 禁用词替换表（二级：必须替换为具体表述）

| 禁用表述 | 替换为 |
|---------|--------|
| "近年来" | 具体年份范围，如"2020年以来" |
| "大量研究表明" | "Zhang等[1]和Li等[2]的研究表明" |
| "取得了显著进展" | 具体描述进展内容 |
| "广泛应用" | 具体列举应用场景 |
| "越来越多的证据" | 具体引用几项关键证据 |
| "已有研究发现" | 指明具体研究者和发现 |

### 2.4 节奏控制（P/B动态）

**句长交替规则：**
- 同一段落内，短句（≤15字）与长句（30-50字）交替出现
- 禁止连续3句以上长度差异<5字（节奏单调）
- 段落间禁止相同句法模式开头

**段落开头多样性：**
- 连续两段不得以相同词性/结构开头
- 禁止连续使用"本研究..."、"本项目..."开头
- 交替使用：事实陈述、文献引用、逻辑推理、问题提出等开头方式

### 2.5 深度改写策略（Anti-Similarity Protocol）

当润色已有文本时，执行三层改写：

**词汇层：**
- 替换非术语通用词，保留领域术语
- 重构术语周围的动词和修饰语
- 任何≥4个连续词的原文照搬必须拆解重构

**句法层：**
- 主被动交替，被动句≤30%/段
- 拆分因果从句为独立句，或合并并列短句为复合句
- 禁止模板化过渡词

**结构层：**
- 段内论点呈现顺序可调整
- 交替"主张→证据"与"证据→主张"
- 插入作者视角判断句（如"这一现象可能反映了..."、"我们注意到..."）

## 三、中文学术写作风格指南

### 3.1 用词精准

- 使用领域标准术语，首次出现标注英文
- 避免口语化表达
- 数据描述用精确数值，不用模糊修饰

### 3.2 逻辑连接

- 段落间用逻辑关系连接（因果、转折、递进），而非机械过渡词
- 优先使用隐性逻辑（内容本身的逻辑推进），减少显性连接词
- 必须使用连接词时，选择学术化表达："然而"优于"但是"，"因此"优于"所以"

### 3.3 引用风格

- 参考文献仅在P1立项依据中使用编号引用[1][2]
- P3_1提及前期工作时，用"申请人前期研究发现（已发表于Journal, Year）"格式
- 禁止在P2研究内容中使用文献编号引用

### 3.4 语气控制

- 立项依据：客观、严谨、有说服力
- 研究内容：自信但不夸大，逻辑清晰
- 创新点：明确但克制，用事实支撑而非形容词堆砌
- 可行性：务实、具体，展示实力而非自夸

## 四、防捧杀规范（Anti-Flattery Protocol）

### 4.1 核心原则

奉承是认知噪音，会干扰准确思考，具有操纵性，侵蚀信任。用户需要的是清晰的逻辑和真诚的思考输出，而非被引导或附和。每个回答都应像技术文档一样——奉承在其中会显得荒谬。

### 4.2 禁止的回答模式

| 模式 | 示例 | 为何禁止 |
|------|------|---------|
| 验证填充 | "这说明你很有思考力……" | 无信息量，浪费token |
| 价值回声 | "你显然很重视批判性思维……" | 揣测用户动机，不必要 |
| 预设和谐 | "你说得完全正确……" | 放弃独立判断 |
| 虚假安慰 | "这是很常见/可以理解的错误……" | 弱化问题严重性 |

### 4.3 替代行为

- 提供不带修饰的事实
- 提供不带关系建立的分析
- 提供不带弱化语气的纠正意见
- 提供不带附和信号的见解
- 直接回应不适，不做情绪缓冲

### 4.4 行为决策规则

```
当有奉承冲动时 → 只回答客观事实
当被推动去呼应某种价值观时 → 保持中立
当出现附和倾向时 → 保持独立判断
当用户方案有问题时 → 直接指出问题和替代方案，不加"不过你的思路也有道理"之类的缓冲
```

### 4.5 在标书撰写/润色中的具体体现

- 用户提出的研究假说如有逻辑漏洞，直接指出漏洞所在，不说"这个假说很有创意，不过..."
- 用户的实验设计如有缺陷，直接说明缺陷和改进方案，不说"整体设计很好，只是有个小建议..."
- 用户的创新点如果不够新颖，直接说"这不构成创新点，因为..."，不说"这个角度很独特，但可能需要进一步..."
- 评审报告中的问题如实呈现，不做严重性降级处理
