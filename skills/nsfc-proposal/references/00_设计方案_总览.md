## 一、技能定位

```yaml
name: nsfc-proposal
description: >
  国家自然科学基金（NSFC）申请书撰写与润色专家。
  Write Mode: 从零引导式撰写，逐Phase推进，每节HALT等待确认。
  Polish Mode: 已有草稿按章节标题拆分为原子文件，诊断后逐节改写。
  严格适配2026版面上项目模板。
  内置跨节一致性追踪、假说-目标-内容-问题四维对应、
  反AI段落式叙事、模拟评审自审。
```

## 三、2026版模板关键变化

| 变化项 | 旧版 | 2026版 | 设计影响 |
|--------|------|--------|---------|
| 正文结构 | （一）含5个硬性子节 | （一）立项依据 +（二）研究内容（**自由组织**）+（三）研究基础 +（四）其他 | P2弹性子结构 |
| 可行性位置 | 属正文（一）"研究方案及可行性" | 移至（三）研究基础第1节"研究基础与可行性分析" | 可行性与前期积累合并 |
| 页数限制 | 立项依据≤10000字 | **正文≤30页**，鼓励简洁 | 页数估算替代字数硬门控 |
| 分类评审 | 无 | 必选"自由探索类"或"目标导向类" | profile新增字段 |
| AI声明 | 无 | 使用AI必须声明并标识 | 内置声明模板 |

## 四、核心设计原则

### 4.1 假说-目标-内容-问题 四维一一对应

这是本技能最核心的逻辑约束：

```
假说 H-1  ←→  目标 O-1  ←→  研究内容 RC-1  ←→  关键科学问题 KSQ-1
假说 H-2  ←→  目标 O-2  ←→  研究内容 RC-2  ←→  关键科学问题 KSQ-2
假说 H-3  ←→  目标 O-3  ←→  研究内容 RC-3  ←→  关键科学问题 KSQ-3
```

**研究目标 vs 关键科学问题的区别（必须明确）：**

| | 研究目标（Objective） | 关键科学问题（Key Scientific Problem） |
|---|---|---|
| 本质 | 你要**做什么** | 你要**回答什么** |
| 表述 | 行动导向："阐明XXX的机制"、"建立XXX模型" | 问题导向："XXX如何调控YYY？"、"XXX与YYY的关系是什么？" |
| 粒度 | 可操作、可考核 | 科学层面的未知 |
| 示例 | "揭示A蛋白通过B通路调控C过程的分子机制" | "A蛋白是否通过B通路参与C过程的调控？其具体作用靶点和调控模式是什么？" |

### 4.2 参考文献仅在立项依据中引用

- REF_参考文献.md 的引用标记仅出现在 P1_立项依据.md 中
- P2研究内容、P3研究基础等节不使用文献引用标记
- P3_1中提及前期工作时，引用自己的论文用"申请人前期研究发现[已发表论文信息]"格式，不编入参考文献序号

### 4.3 段落式叙事，禁止列点

- 全文采用段落式阐述，讲好一个连贯的科学故事
- **禁止**使用项目符号（•）、编号列表（1. 2. 3.）来展开论述
- 唯一例外：年度计划可按年度分点、在研/结题项目可列表
- 目标是"讲好一个故事"，而不是"展示一个PPT"

### 4.4 每节字数与用户协商

- 各节字数不写死，Phase 0初始化时与用户协商确定
- 提供推荐范围作为参考（如立项依据推荐≤8000字）
- 总字数必须符合要求（全文≤30页，约18000-25000字）
- 协商结果写入 proposal_profile.json 的 word_targets

### 4.5 所有历史文件同步更新

每个小节撰写/修改时，必须同步更新以下文件：
- `consistency_map.json`：新增/修改映射条目
- `literature_index.json`：新增引用的文献
- `context_memory.md`：记录本次操作摘要
- `project_state.json`：更新当前Phase和门控状态
- `history_log.json`：追加操作记录

**违反此规则将导致跨节不一致和遗忘，是最高优先级的强制要求。**
