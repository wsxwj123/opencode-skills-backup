## 二、各脚本详细职责

### 2.1 state_manager.py（核心调度器）

**职责：** 项目生命周期管理，所有操作的入口。

```
命令清单：
  init            创建项目目录结构 + 空配置文件
  profile         交互式配置 proposal_profile.json
  load            加载状态（支持 --section / --minimal / --global）
  update          更新状态（merge模式，不覆盖未变更字段）
  snapshot        创建快照（sections/ + data/ + *.json 全量备份到 snapshots/）
  rollback        回滚到指定快照（覆盖当前状态）
  word-count      调用 word_counter.py，按节/全文统计
  page-estimate   调用 word_counter.py，按每页~800字估算
  write-cycle     预写门控：
                    1. 加载当前节已有内容摘要
                    2. 注入 consistency_map 关联条目
                    3. 注入文献上下文（仅P1）
                    4. 分配token预算
                    5. 返回组装好的上下文
  self-review     触发 diagnosis_engine.py 执行评审
  sync-all        强制同步检查：
                    遍历 consistency_map / literature_index /
                    context_memory / project_state / history_log
                    任何一项未更新 → 报错并阻断
                  可选 --auto-fix 自动补齐最小骨架并修复轻微结构问题
```

**关键实现细节：**
- `load --section P1 --minimal`：只读取 P1 的 key_facts（≤10条×80字）+ consistency_map 中 source_section 含 P1 的条目
- `load --global`：读取所有节的 key_facts + 完整 consistency_map + literature_index.metadata
- `snapshot` 使用时间戳命名：`snapshots/20260227_143022_phase1_complete/`
- `rollback` 前自动创建当前状态的安全快照

### 2.3 citation_validator.py（文献卫士）

**职责：** 文献真实性验证 + 引用-索引一致性矩阵检查。

```
  matrix-check    引用-索引一致性矩阵检查
```

**验证流程（每条文献）：**

```python
def verify_entry(entry, search_tools):
    result = {
        "title_match": None,
        "title_similarity": None,
        "doi_valid": None,
        "pmid_match": None,
        "id_cross_match": None,
        "retracted": None,
        "confidence_score": None,
        "needs_manual_review": None,
        "checked_at": datetime.now().isoformat()
    }

    # Check 1: 标题真实性 — 用DOI或PMID反查
    if entry.get("pmid"):
        pubmed_result = search_tools.search_pubmed(entry["pmid"])
        result["title_match"] = fuzzy_match(entry["title"], pubmed_result["title"])
    elif entry.get("doi"):
        semantic_result = search_tools.search_semantic(f"DOI:{entry['doi']}")
        result["title_match"] = fuzzy_match(entry["title"], semantic_result["title"])

    # Check 2: DOI有效性
    if entry.get("doi"):
        result["doi_valid"] = validate_doi_format(entry["doi"])

    # Check 3: PMID对应性
    if entry.get("pmid") and entry.get("doi"):
        result["pmid_match"] = cross_check_pmid_doi(entry["pmid"], entry["doi"])

    # Check 4: 撤稿检查
    result["retracted"] = check_retraction(entry)

    # Check 5: 正文引用核对（需外部传入P1内容）
    # 在调用层面完成

    # 如果出现 title_mismatch/id_mismatch/mcp_stale/source_unreachable
    # 自动进入 manual_review_queue
    return result
```

**引用矩阵检查：**

```python
def matrix_check(p1_content, literature_index, ref_file):
    issues = []

    # 提取P1中所有[N]引用
    p1_refs = extract_citation_numbers(p1_content)

    # 提取索引中所有条目
    index_refs = {e["ref_number"] for e in literature_index["entries"]
                  if "P1_立项依据" in e.get("used_in_sections", [])}

    # 提取REF文件中的条目
    ref_refs = extract_ref_numbers(ref_file)

    # Check 1: P1引用 → 索引必须有
    orphan_citations = p1_refs - index_refs
    if orphan_citations:
        issues.append(f"孤立引用（正文有但索引无）：{orphan_citations}")

    # Check 2: 索引条目 → P1必须引用
    orphan_entries = index_refs - p1_refs
    if orphan_entries:
        issues.append(f"孤立条目（索引有但正文未引用）：{orphan_entries}")

    # Check 3: REF顺序 = P1首次引用顺序
    p1_order = get_first_citation_order(p1_content)
    ref_order = get_ref_file_order(ref_file)
    if p1_order != ref_order:
        issues.append(f"REF顺序与P1首次引用顺序不一致")

    # Check 4: 三方一致
    if p1_refs != index_refs or p1_refs != ref_refs:
        issues.append("P1引用、索引条目、REF文件三方不一致")

    return issues
```

**节奏检查：**

```python
def rhythm_check(text):
    issues = []
    paragraphs = split_paragraphs(text)

    for i, para in enumerate(paragraphs):
        sentences = split_sentences(para)
        lengths = [len(s) for s in sentences]

        # 检查连续3句长度差异<5字
        for j in range(len(lengths) - 2):
            if max(lengths[j:j+3]) - min(lengths[j:j+3]) < 5:
                issues.append(f"第{i+1}段第{j+1}-{j+3}句节奏单调（长度差异<5字）")

    # 检查段落开头多样性
    openers = [get_opener_type(p) for p in paragraphs]
    for i in range(len(openers) - 1):
        if openers[i] == openers[i+1]:
            issues.append(f"第{i+1}段和第{i+2}段开头结构相同（{openers[i]}）")

    return issues
```

## 三、原子文件合并顺序（终稿）

### 3.1 标书正确顺序

```
合并顺序（严格按此顺序，不可调整）：

1.  00_摘要_中文.md
2.  00_摘要_英文.md
3.  B1_预算说明_直接费用.md
4.  B2_预算说明_合作外拨.md
5.  B3_预算说明_其他来源.md
6.  P1_立项依据.md
7.  P2_研究内容.md（或其原子化子文件按原始编号顺序合并）
8.  P3_1_研究基础与可行性分析.md
9.  P3_2_工作条件.md
10. P3_3_正在承担的相关项目.md
11. P3_4_完成基金项目情况.md
12. P4_其他需要说明的情况.md
13. REF_参考文献.md
```

### 3.2 Polish Mode 合并规则

润色模式下P2可能被拆分为多个原子文件，合并时：

```
P2合并顺序 = 按原始编号排序：
  P2_2.1_研究目标.md
  P2_2.2_研究内容.md（如未进一步拆分）
  或：
    P2_2.2.1_纳米药物的构建与表征.md
    P2_2.2.2_纳米药物的体外抗肿瘤活性研究.md
    P2_2.2.3_纳米药物的体内药效学评价.md
  P2_2.3_关键科学问题.md
  P2_2.4_研究方案与技术路线.md
  P2_2.5_特色与创新之处.md
  P2_2.6_年度研究计划.md
```

排序规则：按文件名中的编号数值排序（2.1 < 2.2 < 2.2.1 < 2.2.2 < 2.3）。

### 3.3 合并时的格式处理

```
合并规则：
  1. 每个大节（P1/P2/P3/P4）之间插入分页符
  2. 摘要和预算说明书各自独立分页
  3. 参考文献位于合并文件末尾，与最终标书顺序一致
  4. 保留各节的标题层级
  5. 删除原子文件中的元数据注释（如有）
  6. 统一标点符号（全角中文标点）
```

### 3.4 合并前检查清单

```
合并前必须通过：
  □ sync-all 通过
  □ consistency_map validate 全部通过（至少无ERROR）
  □ citation_validator matrix-check 通过
  □ word_counter count-all 在协商范围内
  □ page-estimate ≤ 30页
  □ humanizer_zh scan-all 无一级违规
  □ L2 full-review 无D级评分
```

任何一项未通过 → 阻断合并 → 输出具体问题 → 修正后重新检查。

### 4.2 回滚规则

- 回滚前自动创建当前状态的安全快照（`pre_rollback_TIMESTAMP`）

- 回滚后 context_memory.md 追加回滚记录
- 回滚不可嵌套（不能回滚到回滚前的安全快照再回滚）
