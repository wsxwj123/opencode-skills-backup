# 05 Write Mode 完整流程

---

## 流程总览

```
Phase 0  项目初始化
Phase 1  立项依据 [P1]
Phase 2  研究内容 [P2]
Phase 3  研究基础 [P3]
Phase 4  其他说明 [P4]
Phase 5  预算说明书 [B1-B3]
Phase 6  中英文摘要 [00]
Phase 7  全文自审与终稿
```

每个Phase结束后执行：snapshot → sync-all → HALT等待用户确认 → 进入下一Phase。

---

## Phase 0: 项目初始化

**目标：** 收集所有必要信息，建立项目骨架。

```
Step 0.1: 收集基本信息
  ├─ 研究方向/课题名称
  ├─ 学科代码
  ├─ 项目类型（面上/青年/重点等）
  ├─ 研究属性（自由探索类 / 目标导向类）
  ├─ 项目周期（默认4年）
  ├─ 经费总额
  └─ 合作单位数量（≤2）

Step 0.2: 收集研究基础信息
  ├─ 申请人前期工作概述
  ├─ 团队成员及分工
  ├─ 已有实验条件/平台
  └─ 在研/结题项目信息

Step 0.3: 协商P2子结构
  ├─ 展示推荐模板（研究假说→目标→内容→关键科学问题→方案→创新→年度计划）
  ├─ 用户可自定义增删改排序
  ├─ 强制保留：特色与创新之处 + 年度研究计划
  └─ 确定假说-目标-内容-问题的数量（如3组）

Step 0.4: 协商各节字数
  ├─ 展示推荐字数范围
  │   ├─ P1立项依据：推荐≤8000字
  │   ├─ P2研究内容：推荐≤8000字
  │   ├─ P3研究基础：推荐≤6000字
  │   ├─ P4其他说明：推荐≤500字
  │   └─ 预算说明书：推荐≤2000字
  ├─ 用户确认或调整
  ├─ 自动校验：各节之和是否在18000-25000总量范围内
  └─ 写入 proposal_profile.json 的 word_targets.user_agreed

Step 0.5: 初始化项目
  ├─ state_manager.py init
  ├─ state_manager.py profile（写入所有配置）
  ├─ 创建目录结构
  ├─ 创建空的 consistency_map.json
  ├─ 创建空的 literature_index.json
  └─ snapshot("phase0_init")
```

**门控：** 所有Step完成 + 用户确认 → 进入Phase 1。

---

## Phase 1: 立项依据 [P1]（最关键，占正文~30-40%）

**目标：** 完成立项依据撰写 + 参考文献库建立。

```
Step 1.1: 文献检索 Round 1
  ├─ 使用 paper-search MCP 检索（见04_文献管理.md）
  │   ├─ PubMed主检索（3-5组关键词）
  │   ├─ Semantic Scholar补充（引用网络）
  │   └─ Google Scholar扩展（中文期刊）
  ├─ 筛选≥30篇核心文献
  ├─ 录入 literature_index.json
  ├─ 执行文献验证流程（全部5项Check）
  ├─ 检查引文指标门控（总数≥30，近5年≥20，中文≥5）
  └─ 不满足 → 补充检索 → 重新验证

Step 1.2: 撰写立项依据
  ├─ write-cycle 加载上下文
  ├─ 按叙事线撰写（段落式，禁止列点）：
  │   ├─ 宏观背景（领域重要性）
  │   ├─ 聚焦具体问题（研究对象）
  │   ├─ 国内外研究现状（已知什么）
  │   ├─ 关键空白/矛盾（不知道什么）
  │   ├─ 本项目切入点（为什么我们能做）
  │   └─ 科学问题凝练（要回答什么）
  ├─ 文献引用使用[1][2]编号，仅在本节使用
  ├─ 反AI风格检查（禁用句式/修辞/词汇）
  └─ HALT → 用户审阅确认

Step 1.3: 注册一致性条目
  ├─ 从P1中提取 SQ（科学问题）条目
  ├─ 写入 consistency_map.json
  └─ 提取 key_facts 写入节级摘要

Step 1.4: 生成参考文献列表
  ├─ 按P1中首次引用顺序生成 REF_参考文献.md
  ├─ 执行引用-索引一致性矩阵检查
  └─ 不一致 → 修正 → 重新检查

Step 1.5: 门控检查
  ├─ word-count → 是否在用户协商范围内
  ├─ 文献全部 verified: true
  ├─ 引用矩阵一致
  ├─ consistency_map 中 SQ 条目已注册
  ├─ sync-all 通过
  ├─ snapshot("phase1_complete")
  └─ HALT → 用户最终确认 → 进入Phase 2
```

---

## Phase 2: 研究内容 [P2]（自由组织，四维对应）

**目标：** 完成研究内容撰写，建立假说-目标-内容-问题四维对应。

```
Step 2.1: 文献检索 Round 2（补充）
  ├─ 针对技术路线中的关键方法检索方法学文献
  ├─ 检索最近6个月最新进展（bioRxiv/medRxiv）
  ├─ 更新 literature_index.json（但不在P2中引用编号）
  └─ 验证新增文献

Step 2.2: 按协商子结构逐节撰写
  ├─ 研究假说（H-1, H-2, H-3...）
  │   └─ 每个假说从P1的科学问题(SQ)逻辑推演而来
  ├─ 研究目标（O-1, O-2, O-3...）
  │   └─ 每个目标对应一个假说，行动导向表述
  ├─ 研究内容（RC-1, RC-2, RC-3...）
  │   └─ 每块内容对应一个目标，详细描述研究工作
  ├─ 拟解决的关键科学问题（KSQ-1, KSQ-2, KSQ-3...）
  │   └─ 每个问题对应一个假说，问题导向表述
  ├─ 研究方案与技术路线（M-1, M-2...）
  │   └─ 每个方法追溯到具体RC
  ├─ 特色与创新之处（IN-1, IN-2...）
  │   └─ 每个创新点追溯到RC和M
  └─ 年度研究计划及预期成果
      └─ 每个RC关联到具体年度

  全程段落式叙事，禁止列点（年度计划按年度分段除外）。
  P2中不使用文献编号引用。

Step 2.3: 注册一致性条目
  ├─ 写入 H / O / RC / KSQ / M / IN 全部条目
  ├─ 建立四维映射关系
  ├─ 执行 consistency_map 验证规则 V-01 ~ V-10
  └─ 任何ERROR级别规则失败 → 修正 → 重新验证

Step 2.4: 门控检查
  ├─ word-count → 是否在用户协商范围内
  ├─ 四维对应完整（H/O/RC/KSQ数量相等且映射完整）
  ├─ consistency_map 验证全部通过
  ├─ sync-all 通过
  ├─ snapshot("phase2_complete")
  └─ HALT → 用户确认 → 进入Phase 3
```

---

## Phase 3: 研究基础 [P3]

**目标：** 完成研究基础四个子节，可行性与前期积累合并论证。

```
Step 3.1: P3_1 研究基础与可行性分析
  ├─ write-cycle 加载：P3_1 + consistency_map中的M和F条目
  ├─ 撰写前期工作积累（段落式）
  │   └─ 提及自己论文用"申请人前期研究发现（Journal, Year）"格式
  ├─ 撰写可行性分析
  │   └─ 每个关键方法(M)必须有可行性证据(F)支撑
  ├─ 撰写风险应对方案
  ├─ 注册 F 条目到 consistency_map
  └─ HALT → 用户确认

Step 3.2: P3_2 工作条件
  ├─ 已有实验条件/平台
  ├─ 缺少的条件及解决途径
  ├─ 注册设备类 F 条目
  └─ HALT → 用户确认

Step 3.3: P3_3 正在承担的相关项目
  ├─ 列表格式（此处允许列表）
  ├─ 说明与本项目的区别和联系
  └─ HALT → 用户确认

Step 3.4: P3_4 完成基金项目情况
  ├─ 结题项目信息
  ├─ ≤500字总结摘要
  ├─ 成果目录（允许列表）
  └─ HALT → 用户确认

Step 3.5: 门控检查
  ├─ word-count → P3总字数在协商范围内
  ├─ consistency_map 中每个M至少有一个F
  ├─ F来源均为P3_1或P3_2
  ├─ sync-all 通过
  ├─ snapshot("phase3_complete")
  └─ HALT → 用户确认 → 进入Phase 4
```

---

## Phase 4: 其他说明 [P4]

```
Step 4.1: 逐项确认5项说明内容
  ├─ 申请人同年申请/参与其他项目情况
  ├─ 具有高级职称的申请人是否作为PI承担过基金项目
  ├─ 申请人是否为博士后
  ├─ 与本项目相关的其他需要说明的问题
  └─ AI使用声明（如 ai_usage_declaration: true）

Step 4.2: 门控
  ├─ sync-all
  ├─ snapshot("phase4_complete")
  └─ HALT → 进入Phase 5
```

---

## Phase 5: 预算说明书 [B1-B3]

```
Step 5.1: 直接费用（B1）
  ├─ 设备费（如有）
  ├─ 业务费
  │   ├─ 材料费 → 三线表
  │   ├─ 测试化验加工费 → 三线表
  │   ├─ 差旅/会议费 → 段落说明
  │   ├─ 出版/知识产权费 → 段落说明
  │   ├─ 劳务费 → 段落说明
  │   └─ 专家咨询费 → 段落说明
  ├─ 预算项追溯到 consistency_map 中的 M 条目
  └─ 更新 budget_table.json

Step 5.2: 合作外拨（B2）
  └─ 如无合作单位则写"无"

Step 5.3: 其他来源（B3）
  └─ 其他经费来源说明

Step 5.4: 门控
  ├─ 预算总额 = profile中的budget_total
  ├─ 三线表格式正确
  ├─ sync-all
  ├─ snapshot("phase5_complete")
  └─ HALT → 进入Phase 6
```

---

## Phase 6: 中英文摘要 [00]

**最后写摘要，基于全文提炼。**

```
Step 6.1: 中文摘要
  ├─ load --global（加载全文摘要）
  ├─ 提炼核心内容，≤400汉字
  ├─ 包含：研究意义、目标、内容、方法、预期成果
  ├─ 关键词与 consistency_map.keywords_trace 一致
  └─ HALT → 用户确认

Step 6.2: 英文摘要
  ├─ 基于中文摘要翻译+润色
  ├─ ≤300英文词
  └─ HALT → 用户确认

Step 6.3: 门控
  ├─ 字数达标
  ├─ 关键词一致
  ├─ sync-all
  ├─ snapshot("phase6_complete")
  └─ HALT → 进入Phase 7
```

---

## Phase 7: 全文自审与终稿

```
Step 7.1: 跨节一致性检查
  ├─ load --global
  ├─ 执行 consistency_map 全部验证规则（V-01~V-10）
  ├─ 检查关键词在各节的出现情况
  └─ 输出不一致项 → 修正

Step 7.2: 引用-索引最终核对
  ├─ 引用矩阵完整检查
  ├─ 文献验证状态全部为 verified
  └─ REF顺序与P1引用顺序一致

Step 7.3: 自审模块（见07_自审与评审模块.md）
  ├─ 执行全维度评审
  ├─ 生成评审报告
  ├─ 根据报告修正
  └─ HALT → 用户审阅评审报告

Step 7.4: 页数估算
  ├─ page-estimate
  ├─ 超30页 → 提示需要精简
  └─ 建议精简的具体位置

Step 7.5: 合并输出
  ├─ 按标书正确顺序合并（见08_合并规则）
  ├─ 生成 申请书_合并.md
  ├─ 生成 申请书_合并.docx（如需要）
  ├─ snapshot("final")
  └─ 完成
```
