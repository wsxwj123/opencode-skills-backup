## 📝 使用流程概览

### 阶段 1：初始化 (30分钟)
```
1. 信息收集 → 2. 环境检查 → 3. 文献分析 → 4. 大纲生成 → 5. 创建项目
```

### 阶段 2：逐章生成 (6-8小时)
```
循环执行 6 次（每章）：
  1. 加载上下文
  2. 分小节生成内容（Markdown）
  3. 插入图表占位符
  4. 转换为 Word（应用样式）
  5. 字数核对（不达标→扩展重写）
  6. 更新状态文件
  7. 用户确认
```

### 阶段 3：后置处理 (1-2小时)
```
1. 生成综述 → 2. 整理参考文献 → 3. 合并所有章节 → 4. 质量自检 → 5. 交付
```

### 独立使用脚本
```bash
# 初始化样式
python3 scripts/setup_styles.py output.docx

# Markdown 转 Word
python3 scripts/markdown_to_docx.py input.md -o output.docx

# 统计字数
python3 scripts/count_words_docx.py thesis.docx --output json

# 合并章节
python3 scripts/merge_chapters.py \
    --input-dir ./02_分章节文档 \
    --output ./完整论文.docx

# 质量检查
python3 scripts/check_quality.py thesis.docx
```
