## 一、会话起点

用户诉求："看看 teacher-paper 这个技能还有什么可优化的地方，总感觉出的卷子不好。"

这是一个开放诊断请求。我先实地读了负责"题目质量"的文件（`anti-ai-guidelines.md`、`exam-templates.md`、`subjects/理科.md`），而不是凭印象。

### 我的初步诊断（4 个结构性问题）
1. **质量保障严重偏科**：语文有 anti-ai 130 行 + exam-templates 224 行的命题工艺；理科只有 50 行、且全是约束、零正向工艺。
2. **内容质量自检不强制**：真实性自检已机器化（exit 2），但"题目好不好"的自检仍是纯 prose，AI 可跳过。
3. **缺整卷视角**：原子化逐题独立，没人审"整张卷子"（考点重复/难度梯度/风格统一/选文撞车）。
4. **anti-ai 工艺只覆盖语文**："怎么让物理题不像 AI 出的"完全没指引。

### 用户澄清（AskUserQuestion 回答）
- **"不好"体现在**（全选 + 补充具体痛点）：
  - 🔴 所有材料总显示"改编自""原创"字眼
  - 🔴 从不问我是否有样卷
  - 🔴 全自动模式只问"出什么卷子"，其他一概不问
  - 题目本身浅、答案解析弱、整卷观感拼凑、某些科目特别差
- **科目范围**：所有科目，逐条完成，先列计划和 todolist
- **推进方式**：先实测出一份卷再定

## 二、根因定位（实测铁证）

### 文档自相矛盾
grep 发现技能文档**6 处**把"改编自《科学之友》"当 `material.source` 的标准写法在教：
- `authoring-workflow.md` 3 处
- `exam-templates.md` 2 处
- `material-sources.md` 1 处

而我**上一轮**加的措辞门禁会拦截"改编"二字 → AI 学文档示例写"改编自" → 撞门禁 → 反复失败/被迫绕过 → 用户天天看到"材料总显示改编自/原创"。

### 实测复现
我造了一个**带合法凭证头的非连材料** + 3 道精心写的非连题，卷面出处按官方示例标"（材料改编自《…报告》）"：
- 标"改编自" → `build` **exit 2 拒绝出卷**
- 改"节选自" → 正常出卷 + 自检表

**铁证**：门禁拦下了一个完全合规的非连题（真实 URL + 凭证 + 题目质量在线），唯一"罪状"是"改编"二字。

### 关键认识
- 我上轮的措辞门禁**过度激进**：把合法的非连"改编"（多则真实材料重组）也拦了。
- "不好"的主因**不是 AI 不会出题**（我实测写的题质量不差），而是**技能自己跟自己打架**，逼 AI 反复妥协。

## 三、用户的"材料真实性"标准（反复澄清，最终定稿）

这是整个会话最核心、且用户**纠正过两次**的需求。最终标准：

> **材料必须真实（禁造假/编造），但允许合理删减。**
> 原文 ABC 三部分：**允许删掉与命题无关的部分，包括中间段（AC 允许）**；但保留的**每段必须逐字取自原文**，禁改字、禁编造、禁打乱原文顺序。

### 纠正轨迹（重要，避免再搞反）
1. 用户最初说"只能首尾连续删，不能 AC（删中段）"。
2. 我据此做了"连续子串门禁"（整块拼接必须是原文连续子串，拒 AC）。
3. 用户**纠正**："不是只能首位删除，而是要保证真实性、禁造假，但允许合理删减，比如原材料跟出题无关的部分可以删掉。" → **AC 应允许**。
4. 我改为：**每段各自是原文逐字子串 + 按原文顺序 + 允许跳过中间段**（`_check_faithful_excerpt` 最终版）。

### 真实性的"两类颗粒度"（用户已接受）
不同学科守法不同，必须分清：
- **A 整段真实**（语文阅读选文/文言/古诗/名著）：每段逐字取自原文，可删无关段，禁改/编造/重排 → `_check_faithful_excerpt` **机器铁证**。
- **B 事实/数据真实**（理科数据/史实/时政/地理数据）：数据/年代/反应式/常量必须真实正确，题目是命制的 → 机器**无法**判定数值对错，靠"情境引子真抓有凭证 + AI 交叉核验 + 人工终审"。
- 用户明确接受 B 类的边界：**机器证不了理科每个数据对错，不假装能**。

## 四、搜索方案的反复（最终：不做脚本，交给 MCP）

这一块走了弯路，用户纠正多次，最终结论稳定。

### 演进过程
1. 我先写了 `search_materials.py`（v3.13.0）：学科×体裁路由 + SearXNG 兜底 + 客户端原生搜索引导。
2. 用户质疑："搜索没必要单独脚本，现在 AI 一般都有搜索能力；没联网搜索能力时单独脚本也没用。"
3. 我先同意删 → 用户提醒"不要那么容易动摇，要有主见" → 我**反驳**：搜索脚本的价值不是"搜索动作"，是"锁权威源"+"无搜索环境兜底"。
4. 用户给出**决定性新论据**："我会用专业搜索 MCP（Tavily/Exa 类，效果好，秒杀脚本）。"
5. 我据此**改判删脚本**，并给出更硬的架构理由 ↓

### 最终结论（v3.15.0）
**搜索交给 AI/MCP，技能不自造搜索脚本。** 架构硬理由：
- 搜索 MCP 由 AI **直接调用**，脚本是子进程，**物理上插不进 AI↔MCP 调用链**。
- 一个插不进主流程的脚本 = 死代码 → 删 `search_materials.py`。

### 但真实性源头约束没丢（责任转移）
- 删脚本 ≠ "源权威性"靠 AI 自觉裸奔。
- `material-sources.md` 升级为「学科×体裁 → 权威域名」清单，喂搜索 MCP 的 `include_domains`。
- SKILL.md Phase 0 强指令："用搜索 MCP/原生搜索 + 🔴必须锁权威域名 + 绝不用作文网/百度文库等二手脏源。"

### 我提出但未落地的"域名白名单校验"
- 我（有主见地）指出：AI 搜到 URL 后**必须走 fetch_web.py 抓取**，这是脚本能掐住的咽喉 → 可在抓取/build 加"权威域名白名单 warn + 自检表标红"，把"源权威性"机器兜底。

## 五、抓取整合（用户反复强调的核心，已实现）

### 用户诉求（明确且反复）
> 把 fetch-everything 的**内容抓取功能**整合进 teacher-paper，**不写死路径**，这样在其他电脑也能用，保证材料真实。

### 实现：兄弟目录动态探测（v3.14.0）
关键事实：teacher-paper 和 fetch-everything 在任何客户端都同处一个 skills 目录、**互为兄弟**：
- Claude Code：`~/.claude/skills/{teacher-paper, fetch-everything}`
- Windows：`C:\Users\baby\.claude\skills\{...}`

`fetch_web.py` 的 `via_engine` 改为按相对路径定位引擎：
```
① <本技能>/../fetch-everything/scripts/fetch_everything.py  ← 用户更新即生效（最新抓取能力）
② 找不到 → 内置副本（别的电脑没装 fetch-everything 时自包含兜底）
③ 都不行 → jina 直连
凭证头仍由 fetch_web.py 写，真实性门禁不变。
```
**全部相对路径，不写死绝对路径**，跨客户端通用。

### 验证
端到端：调兄弟 fetch-everything 引擎抓取古诗文网 4725 字 + 写凭证头通过。

## 八、当前真实性链路（完整图）

```
① 检索：AI 用搜索 MCP（Tavily/Exa，含 include_domains）/ 原生联网搜索
         🔴 必须锁 material-sources.md 的权威域名，禁二手脏源
        ↓ 拿到真实文章 URL
② 抓取：fetch_web.py（兄弟目录动态探测 fetch-everything 最新引擎）抓全文 + 写凭证头
        ↓
③ 门禁（assemble.py build，全部 exit 2 + 各自 --allow-* 降级）：
   溯源门禁(凭证) → 忠实节选门禁(逐字可删·禁改编造重排) → 措辞门禁(节选自)
   → 字数门禁(分板块) → 完整性门禁(坏JSON/题量) → 缺图门禁
        ↓
④ 自检表：build 自动生成「材料真实性自检表.md」，交付须原样转发用户复核
```

门禁开关清单：`--allow-unsourced`/`--allow-excerpt`/`--allow-wording`/`--allow-length`/`--allow-incomplete`/`--allow-missing-figure`。仅用户主动要求或确属误伤时用，且必须告知用户；门禁拒绝 = 返工信号（重抓→重命题→重跑，循环到过）。

## 十一、教训与方法论（本会话提炼）

1. **AI 守规靠机器门禁，不靠 prose**：关键规则必须沉到 build 门禁 exit 2；prose 宣言 AI 会跳过。这是本技能一以贯之的哲学。
2. **跟证据走，不跟人走**：用户说"别动摇"时，正确做法不是固执翻回，也不是顺从，而是看关键事实是否变了（如"通用搜索"→"专业搜索 MCP"，事实变了结论就该变）。
3. **脚本只做"AI 做不了的确定性活"**：搜索是 AI 能力（不造轮子）；抓取+凭证+门禁校验才是非脚本不可的。
4. **不写死路径**：用相对本技能目录的路径定位兄弟技能，跨电脑/客户端通用。
5. **真实性分两类颗粒度**：语文整段可机器铁证，理科数据只能核验+人工终审，别过度承诺。
6. **同步方向单向**：mac `.claude` 权威源 commit + push GitHub，Windows 各客户端拉；不在镜像/其他副本反向 commit（防覆盖丢失）。
