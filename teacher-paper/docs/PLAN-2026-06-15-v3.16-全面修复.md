## 三、推进规则（每 Batch 内）

1. **小步前进**：每 Batch 改完即跑回归（必须 ≥ 之前 62 例）
2. **commit-push 即落地**：每批完成 commit + push GitHub，Windows 端 `git pull` 即可
3. **错位记录**：每批 commit message 列出修复的 Task ID（如 `feat: Batch 1 安全补丁 (Bug-A1/A2/A3/A4/W-1)`）
4. **不走回头路**：发现规划错就在本文件后追加修订，不偷改本节
5. **断点续作**：若中断，下次接续前先 `git log --oneline -5` 确认到哪一批
