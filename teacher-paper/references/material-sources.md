# 联网选材来源与检索方法（全科目）

> 所有抓取统一用 `scripts/fetch_web.py "<url>"`；失败（反爬/登录态）一律提示用户截图或提供原文。
> 学术文献不在此列（按全局规则走 PubMed/OpenAlex）。本表覆盖语文/英语/文科/理科，2026-06 实测过可抓性。

## 〇、全科目素材源·实测速查 + 决策规则（先看这张）

**✅ 可直接抓到正文（已实测）——抓取主力：**

| 科目 | 来源 | 站点 | 适用学段 |
|------|------|------|----------|
| 语文·非连/现代文/文科时政 | 人民网、新华网、中科院科普 | people.com.cn、news.cn、cas.cn | 小学—高中 |
| 英语·阅读/完形 | China Daily（偏难）、VOA Learning English（自带三级分级）、21世纪英文报（小/初/高分级） | chinadaily.com.cn、learningenglish.voanews.com、i21st.cn | 按分级选 |
| 文言文/古诗词 | 古诗文网（**只用 www 主域**）、ctext | www.gushiwen.cn、ctext.org | 小学—高中 |

**⚠️ 抓不到/需特殊处理（不要写进自动抓取链路）：**
- 读者官网 403、意林无官网、三联生活周刊是 SPA、BBC 被墙、Breaking News English 403 → 这些**别自动抓**。
- 微信公众号（读者/意林/博物/三联）→ **只能由用户给单篇链接**再抓，无法列表发现。
- 三联/学科网等 SPA → 需 `fetch-everything`/浏览器渲染，成本高，非必要不用。

**🔴 必须用户提供（铁律下不可抓/不可编造）：**
- 任何"出自某教材某课"的指定选文 → 用户提供原文，**禁止臆造出处**。
- 复杂图（历史地图、地理等值线、电路图、化学结构式、复杂实验装置）→ 用户提供图片（figure 块 `src`）或文字描述。

> **范围说明**：本技能做**全科目**（文科语文/英语/政治·道法/历史/地理 + 理科数学/物理/化学/生物）。
> **文科阅读类**走真实素材+溯源门禁；**理科**按知识点**原创命题**（不抓真题），可画的图自动生成、复杂图用户提供。

## 理科素材与命题（数学/物理/化学/生物）

理科**不抓真题**（题库全付费、搬运违规、答案不可控），而是**按知识点/课标原创命题**：

| 来源 | 用途 |
|------|------|
| 用户提供**样卷** | 解析题型/分值结构 → `--blueprint-file`，最贴合本地卷（强烈推荐）|
| 用户给**知识点/单元范围/教材版本** | 据此原创对应难度的题 |
| 课标知识点（AI 自身掌握）| 无样卷时按学段课标常规题型生成，提示用户核验 |

- **配图**：函数图像/平面几何/数轴/统计图/受力矢量 → `figure` 块写 `kind` 自动画（见 `make_figure.py`）；电路/化学结构/复杂装置 → 用户提供图片 `src`。
- **公式**：简单 Unicode（x²、√、≤、½）；复杂用 `formula` 块（`latex` → mathtext 渲图）。
- **真实性**：公式/常量/化学方程式（配平）/定律/单位/数据必须正确，答案须真能解出、可验证——**不编造"看起来对"的理科内容**。

**一句话决策规则：**
> 文科：新闻/科普/英语分级/公版古籍→自动抓；杂志深度文/公众号→用户给链接；指定教材选文→用户给材料绝不编造。
> 理科：按知识点原创出题；有样卷先解析样卷；可画的图自动画、复杂图用户给。

## 一、文言文（公版古籍，可靠可抓）

| 来源 | 站点 | 说明 |
|------|------|------|
| 古诗文网 | https://www.gushiwen.cn | 搜《世说新语》《古文观止》篇目，有原文+注释+译文 |
| 中国哲学书电子化计划 | https://ctext.org/shi-shuo-xin-yu/zh | 《世说新语》全文，结构化 |
| 古文观止（站内） | gushiwen.cn 搜"古文观止" | 222篇可逐篇取 |

**检索方法**：
```bash
# 例：取《世说新语·德行》篇
python3 scripts/fetch_web.py "https://www.gushiwen.cn/guwen/book_46653FD803893E4F9837F763E182665A.aspx"
```
- 选材标准：篇幅 100-200 字、有完整事件、含 2-3 个可考实词、能与教材形成主题/人物呼应。
- 命题前务必核对原文用字（不同版本有异文），并自拟注释。

> 注：古诗文网务必用 `www.gushiwen.cn`，避开 `so.gushiwen.cn`（被反爬，返回空壳）。

## 四、非连续性文本素材（🔴 必须真实新闻/科普，禁止编造）

非连续性文本的材料**必须摘自真实的新闻报道或科普文章**，不得自行虚构事件、数据、图表。
`assemble.py build` 对含选文的材料文件有硬门禁：缺 `source`+`source_file` 直接拒绝出卷。

### 检索方式（重要：直接抓文章页，不要依赖 RSS 列表）

> 实测结论：国内多数新闻 RSS 已停更或返回旧缓存（新华网科普RSS 404、人民网RSS 返回数月前缓存），
> jina 代理解析 RSS 的 XML 也常丢标题。**最稳的是抓"具体文章页 URL"**，而非订阅 RSS。
> 正确流程：先在门户/搜索里找到一篇合适的近期报道 → 复制其文章页 URL → `fetch_web.py` 抓正文 → 落盘。

### 可靠来源清单（按优先级）

| 类型 | 来源 | 抓取方式 |
|------|------|----------|
| **时政社会新闻** | 人民网 people.com.cn、新华网 xinhuanet.com、央视网 cctv.com、中国新闻网 chinanews.com.cn | 抓**具体报道页URL** |
| **科普/科技** | 科普中国 kepuchina.cn、果壳 guokr.com、中国科普网 kepu.gov.cn、《博物》《科学世界》官网 | 抓文章页 |
| **数据/图表** | 国家统计局 stats.gov.cn、各部委发布会稿、权威媒体的数据新闻 | 抓含图表的报道页 |
| **地方/校园** | 当地日报（如长沙晚报、湖南日报）社会/教育版 | 抓文章页，适合做地方文化情境 |

### RSS 订阅（可用时优先，但多数已不稳，仅作备选）

```bash
# 能抓通就用，抓不到/内容过旧立刻改抓具体文章页：
python3 scripts/fetch_web.py "http://www.people.com.cn/rss/society.xml"   # 人民网社会（时有缓存）
python3 scripts/fetch_web.py "https://www.chinanews.com.cn/rss/society.xml"  # 中新网社会
# RSS 失效是常态，不要硬等——RSS 只用来"发现选题"，正文一律回到文章页抓。
```

### 落盘与命题流程（必做）

```bash
# 1) 抓真实报道正文，存到工程 materials/
python3 scripts/fetch_web.py "<新闻文章页URL>" > "<工程>/materials/非连_新闻-XX.md"
# 2) 据真实材料命题，材料文件 meta 写：
#    "source": "<新闻URL或'XX报XX年X月X日报道'>", "source_file": "非连_新闻-XX.md"
```

### 如何获取「最新」新闻（发现 → 定位 → 抓取）

skill 自带的 `fetch_web.py` 只负责"抓取一个已知 URL"，不负责"发现最新选题"。发现最新报道按以下顺序：

1. **门户栏目页**（最稳）：抓门户的频道/列表页，里面是按时间排的最新报道标题+链接，从中挑一条再抓正文。
   ```bash
   python3 scripts/fetch_web.py "https://www.chinanews.com.cn/society/"   # 中新网社会频道列表
   python3 scripts/fetch_web.py "https://www.kepuchina.cn/"               # 科普中国首页
   ```
2. **若环境装有"内容抓取/热榜检索增强工具"**：可直接搜热点/取热榜，比硬抓门户更准：
   搜关键词或取热榜 → 找到原始新闻链接 → `fetch_web.py` 抓正文。（没有此类工具就跳过，用第1步。）
3. **用户直接给链接**：最省事，B 方向（用户提供）时优先。
4. **判断"够不够新"**：抓到正文后看其中日期；做月考/期末可用近 1-3 个月内的报道，做中考模拟可用近半年的社会热点。过旧或无日期的，换一条。

> 注意：模型训练知识有滞后，**不要凭记忆"编一条最近的新闻"**——必须实际抓取带日期的真实报道。

### 哪些新闻能做考题？选材筛选 7 条标准

抓到候选报道后，逐条过筛，**全部满足才用**：

| # | 标准 | 说明 |
|---|------|------|
| 1 | **真实可溯源** | 有明确出处和日期，非小道消息、非自媒体洗稿 |
| 2 | **价值导向正** | 传递正向价值，无政治敏感、灾难血腥、争议立场 |
| 3 | **适龄** | 话题、词汇初中生能懂，不涉成人/暴力/复杂时政 |
| 4 | **有信息密度** | 含可考的事实、数据、图表或观点，能出"筛选+推断+迁移"三类题 |
| 5 | **可跨学科** | 最好能与数学/科学/历史等结合（长沙非连特色），如数据、原理、统计 |
| 6 | **篇幅可裁** | 能节选/组合成 2-3 则、总长适中（合计约 600-1000 字）|
| 7 | **非烂大街** | 避开人人都用的旧热点；避开与本卷其它板块主题重复 |

- **可对原文节选、删改控制篇幅，但事实、数据、出处必须真实**，并在卷末标注来源。
- 抓不到 → 请用户提供新闻链接或截图，绝不编造。

## 五、抓取失败时的统一回退

`fetch_web.py` 任一来源失败会输出明确提示。此时：
1. 反爬站点（学科网/组卷网/希沃白板/登录态小红书）→ 请用户**截图**，用多模态 Read 识别。
2. 期刊最新内容抓不到 → 请用户提供 **PDF/图片/文字**，走本地 `read_material.py`。
3. 绝不用臆造或拼凑内容充当真实选文。
