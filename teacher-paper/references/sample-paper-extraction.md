# 样卷解析手册（用户提供样卷 → 完整蓝图 JSON）

> 用户给了真题/样卷时，**先把它解析透，再出题**。目标：从样卷扒出**全部可复用的命题参数**，写成一份蓝图 JSON，`assemble.py init --blueprint-file` 一次吃进去——满分、时长、题型、分值、排版、范围全部贴合用户本地卷，不依赖内置默认。**很多本该问用户的信息，样卷里就有，解析出来给用户确认即可，不必空问。**

---

## 第一步：读样卷（按格式分流）

| 样卷形态 | 读法 | 能拿到什么 |
|----------|------|-----------|
| .docx / .pdf（文字版） | `python3 <SKILL_DIR>/scripts/read_material.py "样卷.pdf"` | 全部文字：标题、注意事项、大题小题、选文、出处、分值 |
| 图片 / 扫描件 / 拍照 | 你多模态直接看图；纯文本模型用 `ocr_image.py` 兜底 | 文字 **+ 排版视觉**（密封线、分栏、字体、居中、答题卡格） |
| 排版细节（密封线/字体/分栏） | **纯文本读不出**——必须看 PDF/图片视觉，或直接问用户 | 见下「排版维度」 |

> ⚠️ `read_material.py` 只提纯文本，**会丢掉所有排版信息**（密封线、楷体、居中、分栏、页码）。排版维度务必看样卷视觉（PDF/图片）或向用户确认，不能假设。

---

## 第二步：提取 checklist（逐维度扒，别漏）

### A. 元信息（多在卷头标题、副标题、注意事项栏）
- **学段年级 / 科目 / 考试类型**：卷头标题"XX市2024年九年级语文期末"
- **地区**：卷头城市名、学校名、方言题、地方文化题反推（影响满分/开卷/折算）
- **卷型**（🔴高中关键）：看题型特征判断全国甲/乙/新高考Ⅰ/Ⅱ/地方卷——英语有无读后续写、语文有无信息类文本、数学多选题等是卷型指纹；**卷型≠地区，必须独立确认**
- **总分 / 时长**：卷头"满分120分，考试时间120分钟"或注意事项第一条

### B. 结构：大题 → 小题（→ skeleton + manifest）
- **每个大题**：序号、名称、该大题总分、含几小题 → `skeleton` 的 section 行
- **大题下的板块小标题**（如"（一）非连续性文本阅读（共8分）"）→ `skeleton` 的 sub 行
- **每个小题**：题号、题型、分值、考点、难度 → `manifest` 一行
- 全卷连续编号还是按大题重计？记下编号方式

### C. 阅读素材（逐板块，决定你出卷时抓什么样的真实素材）
对每个阅读板块记录：
- **体裁/题材**：小说/散文/非连/文言/古诗词/名著；乡土/科技/传记…
- **净字数**：数样卷选文的实际字数（中文按字、英文按词）——你出卷时照此长度抓
- **来源出处**：看选文末尾"节选自《XXX》"——记下出处类型（新闻/科普/公版古籍/杂志美文），你出卷时从**同类真实来源**抓，不照搬原文

### D. 排版维度（看视觉，纯文本读不出）
- **密封线 / 座位号**：卷子左侧有无密封装订线 → `sealing_line`
- **页码**：页脚有无"第X页 共Y页" → `page_number`
- **答题方式**：是否配答题卡（注意事项写"答案写在答题卡上"）→ `answer_method`
- **注意事项栏**：原卷的考生须知文字 → `notice`
- 字体（选文楷体）、古诗词居中、分栏——`make_paper.py` 自动按板块处理，**确认有无即可，不用手动设**

### E. 难度 / 特殊项
- 英语：有无**听力**大题（纯文字卷无音频，须三选一，见 SKILL.md 英语特别说明）
- 有无多选题、实验探究题、跨学科题
- 作文：命题/半命题/材料作文、字数下限（统一用横线作答，不出方格纸）

## 第三步：写成蓝图 JSON（字段与 `assemble.py` 完全对齐）

```json
{
  "stage": "高三", "subject": "地理", "region": "全国卷I", "exam_type": "高考",
  "total": 100, "duration": 75,
  "questions": 0,
  "subtitle": "（满分：100分　考试时间：75分钟）",
  "notice": ["1. 全卷满分100分，考试时间75分钟。", "2. 答案写在答题卡上。"],
  "sealing_line": true, "page_number": true,
  "answer_method": "答题卡",
  "exam_scope": "必修一~必修三", "textbook": "人教版", "answer_detail": "详细",
  "novel_len": [800, 1200],
  "skeleton": [
    ["100_sec_选择", "一、选择题（共44分）"],
    ["200_sec_非选择", "二、非选择题（共56分）"],
    ["210_sub_区域", "（一）区域地理综合题"]
  ],
  "manifest": [
    ["101", "q01", "单项选择", 4, "大气环流", "0.65"],
    ["201", "q12", "综合题", 24, "区域可持续发展", "0.55"]
  ],
  "summary": "合计：以用户样卷为准"
}
```

**字段说明（哪些从样卷哪里来、脚本怎么用）**：

| 字段 | 来源（样卷的哪里） | 脚本行为 |
|------|------------------|---------|
| `stage/subject/region/exam_type` | 卷头标题 + 卷型判断 | CLI 未显式指定时**以蓝图为准**（不会被默认成九年级长沙）|
| `total/duration` | 卷头/注意事项 | 写进 meta 与副标题 |
| `questions` | 全卷小题总数 | **填实际题量 → build 校验题量分值；填 `0` → 跳过题量门禁**（结构不确定时用0，但须人工把关）|
| `skeleton` | 大题+板块小标题 | id 含 `_sec_`→大题，否则→板块小标题；数字前缀定排序 |
| `manifest` | 逐小题 | 写进 `00_manifest.md` 命题索引，出卷前给用户逐题确认 |
| `sealing_line/page_number` | 排版视觉 | 渲染密封线/页脚页码 |
| `answer_method` | 注意事项 | 卷面作答 / 答题卡（决定注意事项措辞）|
| `exam_scope/textbook` | 卷头或用户告知 | 写进 meta，约束考点范围 |
| `novel_len` | 数样卷小说选文字数 | build 据此校验选文字数（非九年级默认1000-1500时必填）|
| `notice` | 原卷考生须知 | 不填则按模板自动生成 |

> 排版项（`sealing_line/page_number/exam_scope/textbook/answer_detail`）也可改用 init 的 CLI 参数传（`--sealing-line/--page-number/--scope/--textbook/--answer-detail`）；**CLI 显式参数优先于蓝图**。写进蓝图的好处是一份 JSON 记全、一次 init 到位。

存为 `<工程父目录>/结构.json`，调用：
```bash
python3 <SKILL_DIR>/scripts/assemble.py init "<试卷名>_工程" --blueprint-file "<工程父目录>/结构.json"
```

## 第四步：回填用户确认（必做，别解析完就闷头出题）

把解析出的规格表展示给用户**逐项确认或修正**，再 init：

```
我从样卷解析到：
- 学段科目：高三·地理        卷型：全国卷Ⅰ      地区：（未显式，按全国卷）
- 满分/时长：100分 / 75分钟
- 大题结构：一、选择题44分(11题×4) ；二、非选择题56分(3题)
- 排版：有密封线、有页脚页码、答题卡作答
- 阅读/材料：区域地理综合题含1幅区域图（需你提供，无法自动画）
- 范围：必修一~三（人教版）
以上是否准确？哪项要改？
```

确认后写蓝图 JSON → init → 进入命题。**样卷里读不到的（如本次考试范围、教材版本、地图素材谁提供），在此一并问清。**

## 常见坑

- **别照搬样卷原文当新卷选文**：样卷选文只用来定"体裁/字数/来源类型"，新卷必须按真实性铁律重新抓**同类真实素材**（见 `material-sources.md`），原创题目。
- **卷型判断不准就问用户**：高中卷型指纹不明显时（如只给了部分），直接问"这是哪套卷（全国甲/乙/新高考Ⅰ/Ⅱ/地方）"，不要猜。
- **排版别假设**：没看到样卷视觉就不知道有无密封线/答题卡——要么看图，要么问用户，默认值（页码开、密封线关、卷面作答）仅在用户确认"按默认"时用。
- **题量门禁**：`questions` 填了实际数，build 会卡题量分值是否吻合；拿不准就填 `0` 跳过门禁，但必须人工逐题核对 manifest。
