# Teacher-Paper 教师智能出题技能

> 🔴🔴🔴 **第一铁律：阅读材料禁止编造，只能忠实节选** 🔴🔴🔴
> 非连续性文本、现代文（小说/散文）、古诗词、文言文、名著，**以及政治/历史/地理题里的史料、时政、数据材料**——一切带材料的文科题，其**选文与材料必须取自真实来源**，且只能**忠实节选**（逐字摘录、可删减控篇幅，**禁止重排、改写、编造**；出处一律写"节选自…"，严禁写"改编自"）：
> - **非连续性文本 → 必须摘自真实新闻报道/科普文章**（来源见 `references/material-sources.md`），用 `fetch_web.py` 抓取，**原文落盘到 工程/materials/**。
> - **古诗文/名著 → 必须是真实存在的作品**，用 `fetch_web.py` 从古诗文网/ctext 核验原文，落盘 materials/。
> - **现代文 → 优先选《读者》《意林》等真实美文**；确有把握的教师原创须在 meta.source 显式写 `原创-已声明`。
>
> **`assemble.py build` 有分级硬门禁**（任一不过即拒绝出卷）：
> - 含选文的材料文件 meta 缺 `source`，或非原创却缺 `source_file`/文件不存在 → 拒绝；
> - **source 是网络链接、但 materials 原文无 `fetch_web.py --save` 抓取凭证头 → 拒绝**（堵死"凭记忆默写+补假URL"）；
> - **meta 标 `原创`、卷面选文却写"节选自/选自《…》" → 拒绝**（自相矛盾）；古诗词标原创 → 拒绝（须公版真作）；
> - **题干含『如图/如下图/图中/根据图/如图所示』等字样但 paper 块无 figure 块 → 拒绝**（v3.13.0，堵"AI 写'如图'却不给图"）；
> - 选文段落与 materials 原文逐字比对落空（疑似改写/重排）、来源无法核验、出处含"改编/改写/编译/整理自" → 告警（如实转告用户人工核对）。
> **严禁先编造内容再补一个假来源**——这是命题的红线，宁可流程慢，不可凭空捏造选文。
>
> **题号自动重排（v3.13.0）**：build 时 assemble 按 `items/*.json` 文件名排序后的题文件出现顺序，从 1 递增静默重排 num，同步覆盖 meta.num 与 paper/answer 块里的 num。**AI 写题时 meta.num 可以随便填，最终以文件顺序为准**——文件名 `NNN_qXX.json` 的 NNN 控制顺序（如 `100_q01.json` 在 `200_q01.json` 之前）。重排发生时 build 会 warn 显示原 num → 新 num。
>
> 🔴 **关于 `--allow-unsourced`（强制绕过门禁，慎用）**：build 提供 `--allow-unsourced` 可跳过溯源/完整性门禁强制出卷。**默认绝不使用**；仅在用户**明确知情并要求**时才加，且**一旦使用必须显式告知用户"本卷已绕过真实性门禁，选文未经溯源校验，请自行核对来源"**。**不得为图省事自行加此参数**。
>
> 🔴 **强制门禁逃逸参数清单（v3.13.0）**：
> - `init <绝对路径>` / `init <名> --here` / `init <名> --on-desktop`：**v3.13.0 起 init 必须显式指定位置**，否则硬拒。AI 必须先 AskUserQuestion 问用户「建在当前项目子目录/当前 cwd/桌面」三选一，**绝不可只给工程名让脚本默认建桌面**。
> - `init --accept-fallback`：学段/科目既无样卷又无预设（如七年级数学）时,脚本默认拒绝;用户确认走兜底骨架后才加。
> - `init --listening-mode <omit|as-reading|user-audio>`：英语含听力时**必加**,三选一处理纯文字卷无音频问题;不加则默认拒。
> - `build --allow-empty`：items/ 无任何题时默认拒（避免产出空白 docx）;测试场景才加。
> - `build --allow-missing-figure`：题干含"如图"但 paper 无 figure 块时默认拒（v3.13.0）;**绝不可为图省事直接加**——要么补 figure 块,要么改题干去掉"如图"字样。
> - `build --allow-unsourced`：跳过素材溯源/完整性门禁强制出卷;**默认绝不使用**,用了必须告知用户"已绕过真实性校验"。
> 这几个参数均属"用户明确知情才能用"的逃逸口,**AI 不得为图省事自行加任一**。
>
> 🔴 **理科的真实性铁律**：理科（数学/物理/化学/生物）题目**按知识点原创生成**，不涉及"选文"，故不走上述阅读溯源门禁；但**科学正确性是红线**——公式、物理常量、化学方程式（须配平）、定律、单位、数据必须正确，**答案必须真能解得出、可验证**；自动配图的 `kind` 参数（函数式/坐标/几何点）要与题意、答案自洽。**不确定就核验，绝不编造"看起来对"的理科内容。**

## 核心命题理念

> "以命题促教学，用原创提素养"

本技能以**自包含脚本作为稳定底座**，同时会优先复用当前电脑/当前 AI agent 已有的 Office、MCP、Pandoc、OCR 等能力。任何 AI agent、任何电脑上，都先扫描能力，再选择最稳路径；只有没有稳定 Word 生成后端时，才安装最小兜底依赖。

六个自包含脚本：`setup.py`（环境自检/装依赖）、`read_material.py`（读文档）、`fetch_web.py`（抓网页）、`ocr_image.py`（图片OCR兜底，无识图能力时用）、`assemble.py`（原子化组卷·init/build）、`make_paper.py`（生成 Word）。

> **关于脚本路径（重要，跨工具通用）**：下文命令里的 `<SKILL_DIR>` 代表**本 skill 的实际所在目录**，不是固定路径。不同 AI 工具/电脑上 skill 位置不同，**不要写死**。开始前先确定它：本 SKILL.md 文件所在目录就是 `<SKILL_DIR>`；脚本在 `<SKILL_DIR>/scripts/`。`setup.py` 运行时会用自身位置自动解析并打印出 `skill 目录`，照此替换即可。

**默认场景**：未指定时按**初三/九年级·语文·中考模拟**、长沙中考标准（21题/120分/120分钟）出卷。

### 适用范围（重要）

**全科目，小学一年级到高三**：文科（语文/英语/政治·道法/历史/地理）+ 理科（数学/物理/化学/生物）。

**配图能力分三层**（`make_figure.py` + `figure` 块自动处理）：

| 层 | 图类型 | 怎么来 |
|----|--------|--------|
| ① 自动生成 | 函数图像、平面几何、数轴、统计图(柱/折/饼/散点)、受力·矢量图、**气候图·人口金字塔(地理)** | `figure` 块写 `kind`+参数 → matplotlib 自动画 |
| ② 用户提供 | 电路图、化学结构式/有机式、复杂实验装置、立体几何、历史地图、地理等值线图 | `figure` 块写 `src`(图片路径) → 直接插入 |
| ③ 缺图降级 | 以上都没有 | 渲染成 `［图：alt］` 占位，不阻断出卷，并提示用户补图 |

**公式**：简单的直接用 Unicode 写进题干（x²、√、≤、分数）；复杂的用 `formula` 块（`latex` 字段 → matplotlib mathtext 渲成图片）。

> 自动配图依赖 matplotlib/numpy/sympy（`setup.py` 会检测，缺则提示安装）；电路/化学结构若要自动化需 `pip install schemdraw rdkit`，否则走 ② 用户提供。

### 全学段：结构从哪来（核心机制）

本技能可出**小学一年级到高三、上述文科科目**的卷。试卷结构按以下优先级解析（`assemble.py` 自动处理）：

1. **样卷蓝图（最优先）**：用户给一份本地真题/样卷 → 你读它（`read_material.py`/识图）→ 解析出"大题/题型/分值/时长"写成结构 JSON → `init --blueprint-file <结构.json>`。**中小学无统一标准卷，本地样卷最贴合,优先走这条。**
2. **内置预设**：`presets/` 下已备核心预设（覆盖**高三的 语文/英语/历史/地理/思想政治**），按 `学段_科目[_地区]` 自动命中；科目别名（政治/道法→思想政治）与地区缺省自动回退。**注**：预设文件名用 ASCII（如 `g12_chinese.json`，避免中文文件名跨平台乱码），你只管传中文 `--stage --subject`，脚本内部映射，不用关心文件名。
3. **内置默认**：九年级语文 → 长沙中考标准（不传任何参数即此）。
4. **通用兜底**：以上都没有时，按科目给通用大题骨架（选择/填空/解答…），并提示按样卷补全。

> ⚠️ **不要硬背分值**：同一"七年级数学期末"在不同区县满分题型都不同。**有样卷一定先解析样卷**，预设/兜底只是没有样卷时的近似。
> 🔴 **用户既无样卷、又未报地区/卷型时，必须 AskUserQuestion 反问清楚，不得静默套用预设/兜底回退**——尤其高中，卷型错了整卷题型结构就错。

> 大资料量、需增量修改时，**务必走原子化工作流**（每题一个 json 文件，最后合并），详见 `references/authoring-workflow.md`。这能防止上下文丢失、让改一题不动全卷。

### 命题原则与特征

**命题五原则（仿得像·有根·依标·适度·有据）与四特征（情境/跨学科/不确定性/素养立意）**——长沙九年级语文的完整表述见 `references/exam-templates.md`「命题原则与特征」；其它学段科目对标对应样卷/课标，原则同构（结构仿真题、选材紧扣教材、考点锚课标、创新适度、考查点优先于题型表象）。

### 底线要求（全学段全科目通用）

1. **原创性**：试题必须原创，杜绝改编或抄袭已有真题
2. **规范性**：遵循课标，难度适中，无政治性和知识性错误
3. **去AI感**：严禁AI高频套话和模板化表述（详见 `references/anti-ai-guidelines.md`）
4. **人情味**：题目措辞自然亲切，贴近学生真实生活，像教了10年书的教师出的

## 启动第一步：环境自检（每次开始前必做）

不要假设依赖齐全、不要假设有任何特定工具。**开始任何出卷工作前，先完成两层能力探测**：

1. **Agent 工具探测**：检查当前 AI 客户端是否暴露 Word/Document/Office/Pandoc/Markdown/Spreadsheet/PPT/OCR 相关 MCP、插件或内置工具。
2. **本机能力探测**：运行 `setup.py`，扫描多个 Python 解释器、Python 包、Pandoc、LibreOffice/Office/WPS、OCR、macOS/Windows 系统转换工具。

> MCP / 插件是当前 agent 的会话能力，Python 脚本无法可靠枚举；因此先由 agent 看自己当前有哪些工具，再用 `setup.py` 看本机有什么。

```bash
# 体检（只看不装）：
python3 "<SKILL_DIR>/scripts/setup.py"
# 机器可读体检报告（推荐）：
python3 "<SKILL_DIR>/scripts/setup.py" --json
# 仅当报告显示没有稳定 DOCX 后端，且用户同意时，安装最小兜底依赖：
python3 "<SKILL_DIR>/scripts/setup.py" --install
# 出理科卷（数/理/化/生）且需自动配图/公式时，先装齐理科库（matplotlib/numpy/sympy）：
python3 "<SKILL_DIR>/scripts/setup.py" --install-science
```

处理规则：
- **不要把当前 `python3` 缺包等同于整台电脑没有能力**：`setup.py` 会扫描多个 Python 解释器；agent 也可能有 Word/Pandoc/文档 MCP。
- **python-docx 是最小稳定兜底，不是唯一方案**：已有可用 `docx` Python 后端时，优先用 `make_paper.py`；没有时再看 Pandoc、Office/MCP、系统转换工具。
- **可选依赖**（PDF/PPT/Excel 读取、网页提取增强、图片OCR）缺哪个、只在用到对应素材类型时才需，按需安装。
- **理科库**（matplotlib/numpy/sympy）：一旦确认要出**理科卷且需自动配图/公式**，立即跑 `setup.py --install-science` 装齐，别等到组卷时才发现画不了图；纯文科卷不需要。
- **外部工具**（Pandoc、LibreOffice、Word/WPS、OCR、系统转换工具）有就用、没有不强求。
- 自带脚本若缺失 → 说明 skill 文件不完整，请用户重新获取完整 skill。

### DOCX 后端选择顺序

| 顺序 | 后端 | 使用条件 | 定位 |
|------|------|----------|------|
| 1 | `python-docx` + `make_paper.py` | 任一 Python 解释器可 import `docx` | 最稳定、离线、版式可控的主路径 |
| 2 | Pandoc + `build/content.md` | 有 Pandoc CLI 或 Pandoc MCP | Markdown 中间产物转 DOCX，可配 reference.docx |
| 3 | Word/Document/Office MCP 或内置文档插件 | 当前 agent 暴露相关工具 | 后处理、局部编辑、补图片、转 PDF、复杂文档操作 |
| 4 | LibreOffice / Microsoft Word / WPS | 本机安装且可被 agent 调用 | 格式转换、渲染校验、PDF 输出 |
| 5 | macOS `textutil` / Windows PowerShell | 只有系统自带工具可用 | 低保真 DOCX 兜底 |
| 6 | 安装 `python-docx` | 上述稳定路径都不可用 | 经用户确认后安装最小兜底依赖 |

> 输出仍以 `assemble.py build` 生成的 `build/content.json` 为唯一结构源；`build/content.md` 是给 Pandoc/MCP/Office 后端复用的中间产物。不要让不同后端各自重新组织题目内容。

## 启动第二步：确定工作模式与工程位置（开工前必做）

环境就绪后，**在出题前先问清两件事并写入工程**，避免中途反复打断或自作主张：

> 🔴 **STOP·开工前必须问清以下 1/2，模式二还须问清 3；未问清不得进入 Phase 1。**

### 1. 工作模式（🔴必问·二选一，不得默认）

用 AskUserQuestion 问用户遇到问题时怎么处理：

- **模式一·全程确认**：遇到任何不确定（选哪篇素材、某题难度、补题方向、抓不到内容如何替代等）都**停下来问用户**，不自行决定。
- **模式二·全自动**：除非触及硬门禁（如素材抓不到、依赖装不上），否则**全程自主完成**，最后一次性交付，中途不打断。

> 选模式二时，**必须在开工前把可预见的决策点一次性问清并定好**（见下"决策点清单"），写入工程的 `meta.json`，之后严格据此自动执行，不再逐题打扰。

### 2. 工程位置（🔴必问）

用 AskUserQuestion 问用户：**这份试卷的工程放在哪个文件夹？**
- 用户给了路径 → 在该路径下新建 `<试卷名>_工程` 子文件夹。
- **用户没指定 / 选"默认" → 自动在桌面新建子文件夹**（跨平台：脚本会自动探测真实桌面，兼容 Windows 中文"桌面"、OneDrive 重定向、Linux XDG）。
- 用纯工程名调用 `assemble.py init "<试卷名>_工程"`（不带路径）即触发桌面默认；给绝对路径则建在指定位置。

> 工程是一个独立子文件夹，所有素材、原子题、成卷都在里面，不污染其它位置；断点续作也回到这个文件夹。

### 3. 决策点清单（🔴模式二·STOP：开工前一次性问清并写入 meta.json 的 `decisions`，否则不得自动出题）

| 决策点 | 选项示例 |
|--------|----------|
| 素材来源方向 | 全自动联网 / 用户提供为主 |
| **考试范围·教材版本** | 不限(全册) / 指定单元课次 + 教材版本册次 |
| **各阅读板块·字数** | 按默认表 / 逐块指定（非连/小说/散文/古诗词/文言/名著/英语阅读）|
| **各阅读板块·题材体裁** | 不限 / 逐块指定（如小说乡土、非连科技、文言传记）|
| **各阅读板块·来源** | 用默认来源自动抓 / 用户指定篇目链接 |
| 选材主题倾向 | 不限 / 指定主题（如成长、家国、科技）|
| 作文命题形式·字数下限 | 半命题（默认）/ 命题 / 材料作文；字数按学段（见 Phase1 作文规格）|
| **作答方式** | 卷面作答（默认，无答题卡）/ 答题卡 |
| **卷面工程项** | 页码开(默认) / 密封线·座位号(默认关) |
| **输出格式** | 仅 docx（默认）/ 同时出 PDF |
| **答案解析详略** | 详细含采分点（默认）/ 简略 |
| 抓不到素材时 | 自动换源重试 / 停下问用户 |
| 难度偏好 | 标准0.65-0.70 / 偏易 / 偏难 |
| 名著考查范围 | 按年级默认 / 指定书目 |
| **配图来源·复杂图谁提供** | 自动可画的自动画 / 地图·电路图·化学结构等由用户提供 |

## 执行流程

### Phase 0：资料收集与读取

按资料类型分流处理，**不遗漏任何一份资料/链接**：

**本地文档**（.docx/.pdf/.pptx/.xlsx/.csv/.txt/.md）：
```bash
python3 <SKILL_DIR>/scripts/read_material.py "文件1" "文件2" ...
```
脚本统一输出文本（表格转 `|` 分隔）。若 docx/pdf 内含图片型阅读材料，脚本会提示，需解压后按下方"图片"三级策略识别。

**图片**（截图/拍照/.png/.jpg）——**三级识别策略，兼容不同模型**：
1. **你（模型）本身支持多模态识图** → 直接看图识别（凡支持识图的多模态模型，用其原生识图能力即可）。
2. **你是纯文本模型、无法识图** → 调本地 OCR 兜底：
   ```bash
   python3 <SKILL_DIR>/scripts/ocr_image.py "<图片>" --save "<工程>/materials/截图_XX.md"
   ```
   （脚本自动降级 RapidOCR→PaddleOCR→tesseract；都没装会提示 `pip3 install rapidocr-onnxruntime`）
3. **两者都不行** → 请用户手动誊录图中文字成 .md 放进 `materials/`。

> **无论哪种方式，识别结果都必须存成 markdown 落盘到 `工程/materials/`**，按规律命名（见下"素材命名规范"），不要只在对话里识别完就丢。这样图片素材和网页素材一样可回看、可溯源、可被门禁校验。

**在线链接**（含历年真题网页、新闻、科普文）：
```bash
# 只看不存（预览正文）：
python3 <SKILL_DIR>/scripts/fetch_web.py "<url>"
# 选作阅读素材 → 必须 --save 落盘到 materials/（自动写入抓取凭证头）：
python3 <SKILL_DIR>/scripts/fetch_web.py "<url>" --save "<工程>/materials/非连_来源-简述.md"
```
多策略降级抓取，输出 Markdown 正文。小红书/B站短链自动跟随重定向。
> 🔴 **凡要作为阅读选文的网络素材，必须用 `--save` 抓取落盘**：`--save` 会在文件头写入机器可校验的抓取凭证（url/sha256/时间戳）。`build` 的溯源门禁据此识别——**source 是网络链接、但 materials 文件无此凭证头的，一律判为"凭记忆默写后补假来源"拒绝出卷**。绝不允许凭记忆默写古文/新闻再手写一个 .md 充当原文。

**联网选材来源**（文言文/现代文/作文素材的具体站点与检索方法）见 `references/material-sources.md`。核心：文言文走古诗文网/ctext 公版古籍（《世说新语》《古文观止》节选）；现代文、作文素材按主题检索《意林》《读者》《作文周刊》《意林·作文素材》美文；非连文本取《科学之友》等科普+数据图表。**抓到的素材一律落盘到工程的 `materials/` 目录**（命题时只读当前题相关素材，避免上下文膨胀）；抓不到一律请用户截图或提供 PDF，绝不臆造拼凑选文。

**反爬站点**（学科网、组卷网、希沃白板、需登录的小红书等）：
脚本抓取失败会明确提示。此时立即告知用户："该网站有访问限制，请截图相关内容，我来识别提取。"

**素材命名规范（所有落盘到 `materials/` 的文件统一遵守）**：

```
<板块>_<来源>-<简述>.md
例：非连_中新网-种子库.md   现代文_意林-补碗.md   文言_古诗文网-世说新语德行.md
    截图_用户-真题第3页.md   作文素材_作文周刊-成长.md
```
- 板块前缀：`非连 / 现代文 / 文言 / 诗词 / 名著 / 作文素材 / 截图 / 真题`
- 文件开头建议写两行元信息：`来源：<URL或出处>` 和 `抓取日期：<YYYY-MM-DD>`，方便溯源与门禁回填 `source/source_file`。

**资料读取后**，提取整理：
- 知识点清单（按课标章节归类）
- 可用素材（文段、诗词、图表、数据、新闻）
- 对应教材单元与语文要素
- 可命题方向（哪些素材适合做什么题型）

> **先问一句**：用户有没有"参考样卷/本地真题"？**有 → 让用户给文件，先解析它定结构**（见下"样卷优先"），很多细节就不用再问了；没有 → 按下面逐项确认。已明确的跳过。默认值用于用户答"按默认即可"。

用 AskUserQuestion 逐步确认：

**第一轮·定位**：①学段年级（**默认九年级**；小学1-6 / 初中7-9 / 高一~高三）②科目（**默认语文**；文科：语文/英语/政治·道法/历史/地理；理科：数学/物理/化学/生物。**理科要顺带确认配图**：函数/几何/数轴/统计/受力图可自动画；电路图/化学结构/复杂装置图需你提供图片；**确认出理科卷后立即跑 `setup.py --install-science` 装齐配图库**）③考试类型（**默认中考模拟**；单元/月考/期中/期末/学考/高考模拟）

> 这些参数对应 `assemble.py init` 的 `--stage --subject --region --type --total --duration --questions`，以及结构 JSON（`--blueprint-file`）。**理科配图**：可画的(函数/几何/数轴/统计/受力)用 figure 块写 `kind` 自动生成；电路图/化学结构/复杂装置让用户提供图片走 figure 块 `src`（见 Phase 3 与「适用范围」配图三层表）。

**第三轮·补充项（极易漏，逐项确认；有默认的可直接用默认）**：

| 项 | 为什么必问 | 默认 | 写入 |
|----|-----------|------|------|

**⑩素材来源方向（关键，二选一）**：
- **A. 全自动**：AI 按 `references/material-sources.md` **自行联网抓取真实素材**命题（阅读类走真实性门禁，绝不编造）。
- **B. 用户提供**：用户给材料/链接（**有多少给多少**）。AI 优先用用户素材，缺口再用 A 补齐并告知。

> 默认 A。无论 A/B，**文科阅读类**选文都必须真实可溯源、落盘 `materials/`。**理科不同**：题目按知识点**原创生成**（不抓真题——题库全付费且搬运违规），用户可提供样卷/知识点范围/教材；理科素材方向通常选 B（用户给范围）或按课标知识点自动命题（见 material-sources.md 理科节）。

**样卷优先（强烈推荐，尤其中小学）**：用户给了样卷时，**按 `references/sample-paper-extraction.md`（样卷解析手册）系统提取**，别只扫一眼凭印象：
1. **读**：`read_material.py` 读 docx/pdf 文字；图片/扫描件多模态看图（**排版细节如密封线/字体/分栏纯文本读不出，必须看视觉或问用户**）；
2. **逐维度扒**（手册第二步 checklist）：地区·卷型·总分·时长 / 大题小题题型分值 / 各阅读板块体裁·字数·来源出处 / 排版（密封线·页码·答题方式）/ 听力·多选·作文规格；
3. **写蓝图 JSON**：字段与 `assemble.py` 对齐——`stage/subject/region/exam_type`（CLI 未指定时以蓝图为准）、`total/duration/questions`、`skeleton`、`manifest`、`sealing_line/page_number/exam_scope/textbook/answer_detail`（排版范围项蓝图也支持）、`novel_len`。完整 schema 与示例见手册第三步；
4. **回填确认**（🔴必做）：把解析出的规格表展示给用户逐项确认/修正，样卷读不到的（本次范围、教材版本、地图素材谁提供）一并问清，**再** `init --blueprint-file 结构.json`；
5. 这样满分/题型/分值/排版完全贴合用户本地卷，不依赖内置默认。
> ⚠️ 样卷选文只用来定"体裁/字数/来源类型"，新卷必须按真实性铁律重新抓**同类真实素材**并原创题目，**禁止照搬样卷原文**。

**第四轮·阅读素材逐块确认（关键，动笔写任何阅读题前必做）**：

结构定下后，对照本卷**每一个阅读板块**（非连续性文本 / 小说·散文 / 古诗词 / 文言文 / 名著 / 英语阅读·完形等），用 AskUserQuestion **逐块**问清三件事，并主动给出默认值与**默认来源提示**——不要直接开抓：

1. **字数**（选文净字数/篇幅，超出会影响难度与排版）

#### 选文与排版铁律（阅读题必守，build 有硬门禁）

| # | 规则 | 落地方式 |
|---|------|----------|
| 1 | **选文必须随卷给全文**——任何阅读题，对应选文/材料正文必须写进试卷，否则学生无法作答 | 用 `material` 块且 `paras` 写全文；缺则 build **拒绝出卷** |
| 2 | **非连续性文本**（不是"非文学文本"）选文末尾**标注出处**，右对齐；出处只写"节选自"，禁写"改编自" | `material.source` 填 `（节选自《科学之友》2024年第6期）` 等，脚本自动右对齐；含"改编"会告警 |
| 3 | **小说/文言/古诗/名著**除正文外，须给**标题 + 作者/朝代或出处标注** | `material.title` + `material.author` + `material.source` |
| 4 | **小说选文 1000-1500 字**（净字数） | 越界 build 会告警；过短补充情节，过长精简 |
| 5 | **所有材料的标题/作者都居中**（文言文/小说/散文/非连/作文材料/古诗词概莫能外） | make_paper.py 自动按 material.title/author 居中 |
| 5b | **古诗词正文也整体居中**（每行 5/7 字整齐），不要简单合并成一段 | `material.layout="verse"`，脚本逐句居中 |
| 6 | 其它材料正文（文言文/小说/散文/非连）：**楷体 + 两端对齐 + 首行缩进 2 字** | `material.layout="prose"`（默认），无需手动设字体 |
| 6b | label "【材料一】" 左对齐顶格（宋体）；source 出处右对齐宋体 | make_paper.py 自动处理 |
| 7 | **一篇选文配多道小题**是真题正规做法 | 用 `211_mat_选文.json` 存 material 块，后续 `212_qXX/213_qXX…` 不再重复正文，只写题干/选项；卷面学生只见一篇文章配多题 |

> material 块写法见 `make_paper.py` 顶部文档与 `references/authoring-workflow.md`。
> **忠实节选铁律**：material 选文只能逐字摘录原文、删减控篇幅，**禁止重排重组、改写、编造**；非连多则材料每则须是某真实来源的连续节选；出处一律"节选自…"，build 检出"改编"会告警。
> **引号自动规范**：成卷时脚本会把 `'我的母亲'`（单引号）等自动修正为中文双引号 `“我的母亲”`，但你仍应优先正确书写，别依赖兜底。

**出题核心规则**：

#### 去AI感策略
```
【绝对禁止】
× "同学们，让我们一起来..." / "请认真阅读下面的材料..."（直接给材料）
× "综合以上材料，谈谈你的看法"（太泛，给具体角度和锚点）
× 选项结构完全一致（如四个"XX的精神/品质/品格/态度"）
× 阅读材料是"鸡汤文"或说教文
× 作文题假大空（"以'坚持'为话题"）
× 干扰项随便凑，一眼看出答案

【务必做到】
✓ 题干简洁直接，不废话
✓ 选项长短参差，表述方式各异
✓ 干扰项反映学生真实常见错误（偷换概念、绝对化、强加因果）
✓ 阅读材料有生活质感（家书、采访、新闻特写、真实故事、科普）
✓ 默写有使用情境
✓ 综合性学习有具体活动场景
✓ 作文题从具体小切口进入，贴近学生真实情感
✓ 全卷语言风格统一
```

#### 选材原则
```
文言文：与教材呼应或对比（同主题/同人物类型/同文体），传记类优先
古诗词：选课外但风格手法呼应课内的作品
小说/散文：推荐《读者》《意林》，有故事性和细节，主题正向但忌说教
非连文本：科普+跨学科+图文混合（《科学之友》《博物》等）
综合性学习：校园生活、社会实践、地方文化的真实任务情境
名著：聚焦核心情节与典型形象，覆盖全书关键转折点，避免烂熟片段
```

#### 难度控制
```
基础题（字音字形、默写）：≥0.8，"必得分"
中等题（综合性学习、非连、各选择题）：0.6-0.8
拔高题（文学探究、创意名著、作文）：0.4-0.6
每板块内部由易到难；设送分题与优生拉分题
全卷难度系数 0.65-0.70
```

#### 能力层次递进
```
识记积累 → 理解感悟 → 分析综合 → 运用表达 → 鉴赏评价 → 探究
```

### Phase 3：原子化组卷 → 生成 Word 试卷 + Word 答案解析

**推荐：原子化工作流**（完整流程见 `references/authoring-workflow.md`）。每题独立成 json 文件，最后合并，避免上下文丢失、便于增量改题。

```bash
# 1) 建工程脚手架（目录+meta+manifest+大题分隔文件 + 复制脚本副本到 工程/scripts/）
#    纯工程名→自动建到桌面（跨平台探测）；给绝对路径则建在指定位置；--mode 写入工作模式
#    默认(不传参)=九年级语文长沙；其它学段科目用 --stage/--subject/--region/--type/--total/--duration/--questions
python3 <SKILL_DIR>/scripts/assemble.py init "<试卷名>_工程" --stage 九年级 --subject 语文 --type 中考模拟 --mode 全自动
#    文科示例：--stage 高三 --subject 英语 --type 高考   （命中预设 g12_english.json）
#    理科示例：--stage 九年级 --subject 数学 --type 中考模拟  （通用兜底骨架，建议配样卷）
#    卷面/范围可选参数：--answer-method 卷面作答|答题卡  --sealing-line true  --page-number false
#                       --scope "第一~三单元"  --textbook 统编版  --answer-detail 详细|简略
#    样卷驱动：先把样卷解析成结构 JSON，再 --blueprint-file "<工程父目录>/结构.json"

# 1b) 决策点写入：跨平台稳妥做法 —— 直接用 Write/Edit 把决策写进 <工程>/meta.json 的 "decisions" 字段，
#     不要靠命令行传 JSON（Windows 的 cmd/PowerShell 对引号处理不同，易出错）。
#     mac/Linux 也可用 --decisions-file <json文件> 传入。

# 2) 抓素材入 materials/，逐题写 items/NN_qXX.json（paper+answer+解析+命题意图）

# 3) 合并校验并出卷 —— init 后改用【工程内脚本副本】，不再碰技能本体脚本！
#    自动校验题量/分值、选文完整性、小说字数；生成两个 Word 到 build/（默认带页码）
#    如需 PDF：用 Word/WPS/Pages 打开 docx 后『另存为/导出 PDF』即可（v3.12.0 已移除 --pdf）
python3 "<工程>/scripts/assemble.py" build "<工程>"
```

> **务必把 build 的所有 `[校验告警]`/`[后端]`/figure 降级等输出如实转告用户**——
> 例如小说字数偏短、题量分值不符、抓取替代源、配图降级为占位、用了 `--allow-unsourced` 等，
> 不要只看到"已生成"就报喜，告警与降级都要让用户知情。

> **为什么用工程内副本（重要）**：`init` 会把脚本复制一份到 `<工程>/scripts/`。之后的 build 一律用这份副本，
> 这样即便临场要调脚本，改的也是工程内副本，**永不污染技能本体**，多份试卷工程互不影响。

**出卷后必做（告知位置，不要替用户打开文件）**：build 末尾会打印两个 Word 的**绝对路径**与所在文件夹。
你要把这两个路径清楚转告用户，然后用 AskUserQuestion 问：**"是否帮你打开成卷所在文件夹？"**
- 用户同意 → 执行打开**文件夹**的命令（macOS `open "<build目录>"` / Windows `explorer "<build目录>"` / Linux `xdg-open "<build目录>"`）。
- **只打开文件夹，绝不直接打开 docx 文件**——用户关掉文件后仍能在文件夹里找到，避免"文件在哪"的困惑。

**⚠️ 出卷前确认门禁（必做，不可跳过）**：在运行第3步 `build` 之前，必须先把 `00_manifest.md` 的**全部题目清单**（题号·题型·考点·分值·难度·选材）完整展示给用户，逐题让用户确认或指出要改的题。**用户明确说"可以出卷"后才运行 build**。避免未经确认直接产出成卷。

原子题文件格式与 block 类型见 `authoring-workflow.md` 与 `make_paper.py` 顶部文档。

输出（缺一不可，落在 `build/`）：
1. **学生试卷.docx**——A4，可直接打印，默认带页脚页码「第X页 共Y页」；按 `meta.sealing_line` 可加密封线与座位号；排版按 `references/formatting-rules.md`
2. **参考答案及解析.docx**——含每题答案 + 详细解析 + 评分标准/采分点
3. **content.json / content.md**——结构化源文件与 Markdown 镜像；当 python-docx 不可用时，供 Pandoc、Word MCP、Documents 插件或其它 Office 工具生成/修复 Word。

> 如需 PDF：用 Word/WPS/Pages 打开 docx，"另存为/导出 PDF" 即可——v3.12.0 起本技能不再内置 LibreOffice 转换（兼容性差、字体丢失常见，外部工具更可靠）。

> 答案解析要包含：客观题答案+错因，主观题采分点（"答出X点给X分"），作文评分等级标准，命题意图与教材关联。
> 单题快速生成也可直接用 `make_paper.py content.json`（不走工程目录），但多资料/需迭代时一律用 assemble 原子化流程。

### Phase 4：审题迭代

生成后分板块展示给用户，主动询问：
- "需要替换或修改哪道题？"
- "各板块难度是否合适？"
- "选文/素材是否满意？"

用户提意见后**只修改对应的 `items/NN_qXX.json`**，重跑 `assemble.py build` 重新出卷，其余题零改动、零风险，循环到满意。manifest（`00_manifest.md`）随时记录每题状态，支持中断续作。

#### 交付时必须告知用户（免责与人工待办，逐条说清）

1. **文件位置**：两份 Word 的绝对路径 + 所在文件夹，并询问是否打开**文件夹**（不直接开文件）。
2. **本卷用了哪些素材来源**（URL/出处/抓取日期），便于用户复核。
3. **触发的告警**：小说字数偏短/偏长、题量分值不符、抓取换源、配图降级占位、是否用了 `--allow-unsourced`——如实转达。
4. **需人工补的内容（待办清单）**：名著须**核对书目版本**；历史地图/地理图表/英语听力音频需用户提供；figure 占位 `［图：…］` 处需补图。
5. **请用户终审**：真实素材经删改、AI 命题可能有疏漏，**事实/数据/出处/答案务必请用户人工校对**；难度系数为命题估计值（非实测）。
6. **版权提醒**：选用的真实新闻/美文仅供课堂测评，正式公开使用请用户自行确认版权。
7. **引号已自动规范**（单引号误用→中文双引号），如与原文有别属正常。

## 特殊处理

### 小红书/社交媒体
提取教学相关内容（知识点、易错题、答题技巧），忽略营销，标注来源。

### 希沃白板课件
提取知识结构、重难点标注，利用课件逻辑辅助知识点覆盖。

### 历年真题参考
分析命题规律与考查重点；新题须区别于原题（换素材/换角度），严禁搬运；可模仿命题风格与难度梯度；避开学生过熟的经典片段。

### 作文命题要点
半命题为稳妥之选；补题宜为有抽象性的名词或名词短语；导语3-5句启发而非限制；可参考"作文三步法""五笔法"（详见 anti-ai-guidelines.md）。

### 古诗文命题避坑
翻译选有语境铺垫的句子；简答给要点而非死答案；至少一题考"怎么写"（手法/表达效果）；断句题（第16题）给四种断法，干扰项错在切错主谓/动宾边界。

### 名著命题避坑
选段须涉关键转折点；选项/问题关联前后情节，不读全书不能答；简答须结合整本书。

### 英语科目特别说明（听力·作文·字数）
- **听力（v3.12.0 起脚本硬门禁）**：本技能出**纯文字 Word 卷，不含音频**。检测到英语含听力大题时 `init` **默认硬拒**，必须 AskUserQuestion 三选一后用 `--listening-mode` 显式重跑：
  - `--listening-mode omit`        省略听力（总分下调或并入阅读/作文，须告知用户改后总分组成）
  - `--listening-mode as-reading`  附听力文字稿当阅读题（保留分值，文字稿须来自真实题源）
  - `--listening-mode user-audio`  用户自备音频（本卷只出题干选项，提醒用户另发音频）
  脚本会把选择写入 `meta.listening_mode`,供后续题目生成参考。**不要默默生成一个无法播放的"听力"大题。**
- **英语作文**：按学段定类型与词数（如初中书面表达 ≥80 词、高考应用文 ≥100 词 / 读后续写约 150 词），不要套用语文"600字"。
- **字数统计**：英语选文/作文按**词**计；`build` 的字数校验对英语已自动按词统计，但默认 1000-1500 区间是中文规格，英语需在蓝图 `novel_len` 显式给词数区间才校验。

### 理科特别说明（数学·物理·化学·生物）

理科按知识点**原创命题**（不抓真题），配图分自动/用户提供两类，公式分 Unicode/图片两类。

> 理科情境一般直接写进题干 `text`，**不用 material 选文块**（material 块会触发阅读溯源门禁）；若确需 material 块承载情境素材，`meta.source` 写 `原创-已声明` 即可通过门禁。

**配图——能自动画的写 `kind`（matplotlib 自动生成；几何题强烈推荐用 svg）**：
```json
{"type":"figure","kind":"function","funcs":["x**2-2*x-3"],"xrange":[-3,5],"alt":"抛物线","width_cm":7}
{"type":"figure","kind":"number_line","min":-3,"max":3,"points":[{"x":1,"label":"a","filled":true}],"intervals":[{"from":1,"to":3}]}
{"type":"figure","kind":"bar","x":["甲","乙","丙"],"y":[3,5,2],"ylabel":"人数"}   // 还有 line/pie/scatter
{"type":"figure","kind":"vector","vectors":[{"start":[0,0],"end":[3,0],"label":"F1"},{"start":[0,0],"end":[0,2],"label":"F2"}],"alt":"受力图"}
```

🔴 **几何题（三角形/圆/平行线/含标注）→ 用 `kind="svg"` AI 直接写 SVG，不要用 `kind="geometry"`**：matplotlib 的 geometry 出图丑且坐标常算错；AI 自己写 SVG 字符串，坐标完全可控，样式接近课本插图：
```json
{"type":"figure","kind":"svg","alt":"Rt△ABC","width_cm":6,
 "svg":"<svg xmlns='http://www.w3.org/2000/svg' width='240' height='200'><line x1='30' y1='170' x2='210' y2='170' stroke='black' stroke-width='2'/><line x1='210' y1='170' x2='210' y2='30' stroke='black' stroke-width='2'/><line x1='30' y1='170' x2='210' y2='30' stroke='black' stroke-width='2'/><rect x='190' y='150' width='20' height='20' fill='none' stroke='black' stroke-width='1.5'/><text x='15' y='185' font-size='18' font-family='serif'>A</text><text x='215' y='185' font-size='18' font-family='serif'>C</text><text x='215' y='25' font-size='18' font-family='serif'>B</text></svg>"}
```
SVG 写法要点：坐标系原点在左上、y 轴向下；线条 stroke-width 1.5~2 较清晰；中文标注用 `<text font-family='serif'>` 或 `font-family='宋体,SimSun,serif'`；直角符号用 `<rect>` 套在角点;line/polygon/circle/text/path 都支持。**走 SVG = AI 自己算几何=结果可控**。

`kind="geometry"`（matplotlib 老路径）保留兼容,**新题一律改用 svg**。
**配图——画不了的让用户给图，写 `src`**（电路图/化学结构式/复杂装置/立体几何）：
```json
{"type":"figure","src":"materials/电路图-1.png","alt":"串联电路","width_cm":7}
```
> 缺图（既无 src 又非可画 kind）→ 自动降级 `［图：alt］` 占位，并在交付时提示用户补图。

**公式**：
```json
// 简单：直接 Unicode 写进题干文本（无需块）： "解方程 x²-5x+6=0"
// 复杂：formula 块（latex → mathtext 渲成图片）
{"type":"formula","latex":"x=\\frac{-b\\pm\\sqrt{b^2-4ac}}{2a}","alt":"求根公式"}
```

**命题避坑**：① 化学方程式必须配平、标条件/状态；② 物理给全已知量与单位，答案可解；③ 数学几何题图形参数与题干数据一致；④ 生物多为文字+表格，遗传题给清楚亲本基因型；⑤ 每题答案自己先解一遍，确保唯一/可验证。

### 政治·历史·地理特别说明
- **时政时效（政治/道法）**：时政材料务必抓**近 1-3 个月**真实报道（月考期末）或近半年热点（学考），抓取后核对日期，过旧即换；绝不凭记忆编时政。
- **教材版本/册次**：政史地考点强绑定教材版本与册次，务必先与用户确认（见第三轮补充项）。
- **图**：地理的**气候图/人口金字塔/统计图可自动画**（`figure` 块 kind=climate/pyramid/bar/line/pie）；**地图/政区图/等值线图/文物图/历史地图无法自动画**，让用户提供图片走 `figure` 块 `src`，或题干文字描述；缺图降级 `［图：…］` 占位并告知用户补。
- **史料/数据真实性**：史料原文走 ctext/古诗文网核验，**用 `material` 块承载以纳入溯源门禁**，只忠实节选、出处写"节选自"；题干内嵌的数据/史实也必须真实，不可编造（溯源硬门禁只校验 `material` 选文块，纯数据题需你自觉把关）。
